Linux 新服务器初始化清单

拿到一台新 Linux 服务器后，先完成用户、SSH、防火墙和更新配置。

新服务器不要急着部署应用，先把基础安全和维护配置做好。下面的顺序适合 Debian、Ubuntu、Rocky Linux 等常见发行版。

创建普通用户

避免长期直接使用 root。创建普通用户后，把它加入 sudo 或 wheel 组，再用这个用户完成后续操作。

配置 SSH 登录

优先使用密钥登录，确认新用户可以登录后，再考虑关闭密码登录。修改 SSH 配置前保留一个已登录窗口，避免把自己锁在门外。

更新系统

先安装安全更新和常用工具，例如 curl、vim、git、htop。生产服务器建议记录每次大版本升级。

开启防火墙

只放行正在使用的端口。比如 SSH、HTTP、HTTPS，其他端口按服务需要再逐个开放。
