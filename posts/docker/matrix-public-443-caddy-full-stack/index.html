机密：Matrix Synapse 全套公网 443 + Caddy 生产搭建手册

从一台 Alpine VPS 手搓 Matrix Synapse public-443 生产部署：DNS、目录、密钥、Caddy、Synapse/Postgres、Element Web/Call、LiveKit、JWT、coturn、ntfy、synapse-admin 和验证流程。

这篇只写一件事：一台新的 Alpine VPS，照着步骤从零搭出可用的 Matrix Synapse public-443 生产栈。没有完整安装步骤的组件不放进本文，避免小白照着做到一半发现缺口。

  • Synapse + Postgres
  • Element Web
  • Element Call
  • LiveKit
  • matrix-livekit-jwt-service
  • coturn
  • ntfy
  • synapse-admin

照着做时，把域名、IP、账号名替换成自己的即可；这里保留当前生产机器的域名和目录，方便直接复刻。

本文示例机器是 192.227.236.209，普通用户是 jay，不设置 root 密码，平时用本地 SSH key 登录后 sudo -i 或 sudo -n sh -c 切 root。生产运行目录统一放在：

/root/data/docker_data/

这个目录是 root 管理的。jay 加入 Docker 组只代表可以操作 Docker，不代表应该把生产 secret 和数据库放到普通用户家目录里。

从零搭建总流程

如果换一台新 Alpine VPS，按这个顺序做，不要跳着来：

  1. 域名 DNS 先指到服务器公网 IP。
  2. Alpine 安装 Docker、Compose、Caddy 所需基础工具。
  3. 创建 /root/data/docker_data 生产目录和 /root/data/docker_data/_secrets 密钥目录。
  4. 生成所有密码、TURN secret、LiveKit key、Synapse registration secret。
  5. 先启动 Synapse + Postgres，确认 matrix.jgaga.tk 基础接口正常。
  6. 再启动 Caddy，把 /.well-known、Synapse、Element Web、Element Call、LiveKit、JWT、ntfy、synapse-admin 接到公网 443。
  7. 启动 coturn。TURN 不能走 Caddy，DNS 必须是 DNS only。
  8. 最后启动 synapse-admin。
  9. 用 curl、Element 登录、Element Call 通话、synapse-admin 登录逐项验收。

下面所有服务器命令默认已经执行过 sudo -i，也就是在 root shell 里操作。看到 cat > /path/file <<'EOF' 这种块时，意思是直接复制整段命令，它会新建或覆盖对应文件，直到最后一行 EOF 结束。

DNS 准备

Cloudflare 里加这些记录，全部指向 192.227.236.209：

jgaga.tk                 A 192.227.236.209
matrix.jgaga.tk          A 192.227.236.209  proxied off 或 on 均可，建议 Matrix API 先 DNS only 排障
element.jgaga.tk         A 192.227.236.209
call.jgaga.tk            A 192.227.236.209
livekit.jgaga.tk         A 192.227.236.209  DNS only
matrix-jwt.jgaga.tk      A 192.227.236.209
turn.jgaga.tk            A 192.227.236.209  必须 DNS only
ntfy.jgaga.tk            A 192.227.236.209
synapse-admin.jgaga.tk   A 192.227.236.209

如果这些 HTTP/HTTPS 子域名开 Cloudflare 橙云代理，Cloudflare 的 SSL/TLS 模式必须使用 完整（严格）。不要用 灵活，否则 Cloudflare 会用 HTTP 回源，Caddy 又会把 HTTP 自动 308 到同一个 HTTPS 地址，浏览器看到的就是重定向循环，表现为 element.jgaga.tk 打不开。

turn.jgaga.tk 必须 DNS only，因为 coturn 是 UDP/TCP TURN 服务，不是 HTTP 网站。Cloudflare 橙云代理不了 TURN。

服务器上先确认 DNS 已生效：

getent hosts matrix.jgaga.tk
getent hosts turn.jgaga.tk

Alpine 基础环境

登录机器：

ssh jay@192.227.236.209
sudo -i

安装基础包：

apk update
apk add docker docker-compose-plugin openssl curl jq bash python3 py3-pip sqlite grep sed coreutils shadow
rc-update add docker default
rc-service docker start

如果普通用户要能执行 Docker：

addgroup jay docker

重新登录后生效。生产目录仍然放 /root/data/docker_data，不是放 /home/jay。

创建目录

install -d -m 755 /root/data/docker_data
install -d -m 700 /root/data/docker_data/_secrets

for d in \
  caddy synapse coturn livekit-server matrix-livekit-jwt-service \
  element-web element-call ntfy synapse-admin
do
  install -d -m 755 "/root/data/docker_data/$d"
done

/root/data/docker_data 就是这套服务的运行目录。手动安装时只维护这一份目录，避免一套配置、多处副本看不清。

生成密钥和密码

真实值都放 _secrets，不要散落在 compose 里。

install -d -m 700 \
  /root/data/docker_data/_secrets/synapse \
  /root/data/docker_data/_secrets/synapse-admin \
  /root/data/docker_data/_secrets/coturn \
  /root/data/docker_data/_secrets/livekit

openssl rand -hex 32 > /root/data/docker_data/_secrets/synapse/registration_shared_secret
openssl rand -hex 24 > /root/data/docker_data/_secrets/synapse/postgres-password
openssl rand -hex 32 > /root/data/docker_data/_secrets/coturn/static-auth-secret
openssl rand -hex 16 > /root/data/docker_data/_secrets/livekit/api-key
openssl rand -hex 32 > /root/data/docker_data/_secrets/livekit/api-secret
openssl rand -base64 24 > /root/data/docker_data/_secrets/synapse-admin/admin-password

printf 'admin2\n' > /root/data/docker_data/_secrets/synapse-admin/admin-user
chmod -R go-rwx /root/data/docker_data/_secrets
chown -R root:root /root/data/docker_data/_secrets

access_token 不是这一步生成。它要等 Synapse 跑起来、管理员账号创建好以后，通过登录接口拿到，再写入：

/root/data/docker_data/_secrets/synapse-admin/access_token
/root/data/docker_data/_secrets/synapse-admin/device_id

获取管理员 access_token

Synapse 跑起来并创建 admin2 后，用 Admin 密码登录一次客户端 API，拿到 access_token 和 device_id：

admin_pass="$(cat /root/data/docker_data/_secrets/synapse-admin/admin-password)"
curl -fsS -X POST 'http://127.0.0.1:8008/_matrix/client/v3/login' \
  -H 'Content-Type: application/json' \
  -d "$(jq -nc --arg user 'admin2' --arg pass "$admin_pass" '{
    type:"m.login.password",
    identifier:{type:"m.id.user", user:$user},
    password:$pass,
    initial_device_display_name:"matrix-admin"
  }')" >/tmp/admin-login.json

jq -r '.access_token' /tmp/admin-login.json > /root/data/docker_data/_secrets/synapse-admin/access_token
jq -r '.device_id' /tmp/admin-login.json > /root/data/docker_data/_secrets/synapse-admin/device_id
chmod 600 /root/data/docker_data/_secrets/synapse-admin/access_token \
  /root/data/docker_data/_secrets/synapse-admin/device_id
rm -f /tmp/admin-login.json

验证 token：

TOKEN="$(cat /root/data/docker_data/_secrets/synapse-admin/access_token)"
curl -fsS -H "Authorization: Bearer $TOKEN" \
  'http://127.0.0.1:8008/_synapse/admin/v2/users?limit=1' | jq .

这个 token 以后可以给只读检查脚本或管理工具使用；如果只是网页登录 synapse-admin，用管理员密码也可以。

Synapse + Postgres

目录：

/root/data/docker_data/synapse/docker-compose.yml
/root/data/docker_data/synapse/data/homeserver.yaml
/root/data/docker_data/synapse/postgres/

先生成 Synapse 基础配置：

cd /root/data/docker_data/synapse
install -d -m 755 data postgres
docker run --rm -it \
  -v "$PWD/data:/data" \
  -e SYNAPSE_SERVER_NAME=jgaga.tk \
  -e SYNAPSE_REPORT_STATS=no \
  matrixdotorg/synapse:latest generate

写 docker-compose.yml。如果这个文件不存在，就用下面命令创建：

cat > /root/data/docker_data/synapse/docker-compose.yml <<'EOF'
services:
  postgres16:
    image: postgres:16-alpine
    container_name: postgres16
    restart: unless-stopped
    environment:
      POSTGRES_DB: synapse
      POSTGRES_USER: synapse_user
      POSTGRES_PASSWORD_FILE: /run/secrets/postgres_password
      POSTGRES_INITDB_ARGS: "--encoding=UTF8 --locale=C"
    secrets:
      - postgres_password
    volumes:
      - ./postgres:/var/lib/postgresql/data

  synapse:
    image: matrixdotorg/synapse:latest
    container_name: matrix-synapse
    restart: unless-stopped
    depends_on:
      - postgres16
    ports:
      - "127.0.0.1:8008:8008"
    volumes:
      - ./data:/data

secrets:
  postgres_password:
    file: /root/data/docker_data/_secrets/synapse/postgres-password
EOF

修改 data/homeserver.yaml，关键项如下。不要整段照搬覆盖，按字段改：

cd /root/data/docker_data/synapse
cp data/homeserver.yaml data/homeserver.yaml.bak.$(date +%Y%m%d%H%M%S)
vi data/homeserver.yaml
server_name: "jgaga.tk"
public_baseurl: "https://matrix.jgaga.tk/"

listeners:
  - port: 8008
    tls: false
    type: http
    x_forwarded: true
    bind_addresses: ['0.0.0.0']
    resources:
      - names: [client, federation]
        compress: false

database:
  name: psycopg2
  args:
    user: synapse_user
    password: "这里填 /root/data/docker_data/_secrets/synapse/postgres-password 的内容"
    database: synapse
    host: postgres16
    cp_min: 5
    cp_max: 10

enable_registration: false
registration_shared_secret: "这里填 /root/data/docker_data/_secrets/synapse/registration_shared_secret 的内容"

Synapse 的 homeserver.yaml 不支持 Docker secret 自动展开，所以数据库密码和 registration secret 最终会出现在这个文件里。这个文件必须留在 root 管理的生产目录，不要复制到公开仓库或其它无关目录。

启动：

cd /root/data/docker_data/synapse
docker compose up -d
docker logs --tail=80 matrix-synapse
curl -fsS http://127.0.0.1:8008/_matrix/client/versions | jq .

创建管理员用户：

admin_pass="$(cat /root/data/docker_data/_secrets/synapse-admin/admin-password)"
docker exec -it matrix-synapse register_new_matrix_user \
  -c /data/homeserver.yaml \
  -u admin2 \
  -p "$admin_pass" \
  -a \
  http://127.0.0.1:8008

Caddy 反代

目录：

/root/data/docker_data/caddy/docker-compose.yaml
/root/data/docker_data/caddy/Caddyfile

docker-compose.yaml：

install -d -m 755 /root/data/docker_data/caddy/data /root/data/docker_data/caddy/config
cat > /root/data/docker_data/caddy/docker-compose.yaml <<'EOF'
services:
  caddy:
    image: caddy:2-alpine
    container_name: caddy
    restart: unless-stopped
    network_mode: host
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile:ro
      - ./data:/data
      - ./config:/config
EOF

Caddyfile 核心结构。第一次安装时直接创建这个文件：

cat > /root/data/docker_data/caddy/Caddyfile <<'EOF'
(matrix_client_cors) {
  # Matrix 客户端会从 Element Web 跨域读取这些发现端点；缺少 CORS 会被浏览器误判为 homeserver 不可达。
  header Access-Control-Allow-Origin "*"
  header Access-Control-Allow-Methods "GET, HEAD, OPTIONS"
  header Access-Control-Allow-Headers "Origin, X-Requested-With, Content-Type, Accept, Authorization"
}

matrix.jgaga.tk {
  encode zstd gzip

  handle /.well-known/matrix/client {
    import matrix_client_cors
    respond `{"m.homeserver":{"base_url":"https://matrix.jgaga.tk"},"org.matrix.msc4143.rtc_foci":[{"type":"livekit","livekit_service_url":"https://matrix-jwt.jgaga.tk"}]}` 200
  }

  handle /.well-known/matrix/server {
    respond `{"m.server":"matrix.jgaga.tk:443"}` 200
  }

  @matrixrtc_transports path /_matrix/client/unstable/org.matrix.msc4143/rtc/transports /_matrix/client/v1/rtc/transports
  handle @matrixrtc_transports {
    header Content-Type application/json
    import matrix_client_cors
    # Element 系列客户端有时不会带 Authorization 访问 MatrixRTC transport discovery；这里直接返回静态 LiveKit 入口。
    respond `{"rtc_transports":[{"type":"livekit","livekit_service_url":"https://matrix-jwt.jgaga.tk"}]}` 200
  }

  @client_versions path /_matrix/client/versions
  handle @client_versions {
    header Content-Type application/json
    import matrix_client_cors
    # Synapse 版本接口未必声明 msc4143；补 capability 可以避免 Element Web 误判通话不可用。
    respond `{"versions":["v1.1","v1.2","v1.3","v1.4","v1.5","v1.6","v1.7","v1.8","v1.9","v1.10","v1.11","v1.12"],"unstable_features":{"org.matrix.msc4143":true}}` 200
  }

  handle /_matrix/* {
    reverse_proxy 127.0.0.1:8008
  }

  handle /_synapse/* {
    reverse_proxy 127.0.0.1:8008
  }

  handle /* {
    respond 404
  }
}

jgaga.tk {
  header Content-Type application/json

  handle /.well-known/matrix/client {
    import matrix_client_cors
    respond `{"m.homeserver":{"base_url":"https://matrix.jgaga.tk"},"org.matrix.msc4143.rtc_foci":[{"type":"livekit","livekit_service_url":"https://matrix-jwt.jgaga.tk"}]}` 200
  }

  handle /.well-known/matrix/server {
    respond `{"m.server":"matrix.jgaga.tk:443"}` 200
  }

  respond 404
}

http://element.jgaga.tk {
  encode zstd gzip
  # Cloudflare Flexible 回源会用 HTTP；这里避免 Caddy 自动 308 到同一 HTTPS 地址造成浏览器重定向循环。
  # 正常生产仍建议把 Cloudflare SSL/TLS 设为“完整（严格）”，这个块只是兜底兼容。
  @element_runtime_config path / /index.html /config*.json /manifest.json /version /sw.js
  header @element_runtime_config Cache-Control "no-store"
  @element_static_assets path /bundles/* /fonts/* /icons/* /img/* /vector-icons/*
  header @element_static_assets Cache-Control "public, max-age=31536000, immutable"
  header X-Content-Type-Options "nosniff"
  reverse_proxy 127.0.0.1:8181
}

element.jgaga.tk {
  encode zstd gzip
  @element_runtime_config path / /index.html /config*.json /manifest.json /version /sw.js
  header @element_runtime_config Cache-Control "no-store"
  @element_static_assets path /bundles/* /fonts/* /icons/* /img/* /vector-icons/*
  header @element_static_assets Cache-Control "public, max-age=31536000, immutable"
  header X-Content-Type-Options "nosniff"
  reverse_proxy 127.0.0.1:8181
}

call.jgaga.tk {
  encode zstd gzip
  reverse_proxy 127.0.0.1:8081
}

livekit.jgaga.tk {
  encode zstd gzip
  reverse_proxy 127.0.0.1:7880
}

matrix-jwt.jgaga.tk {
  encode zstd gzip
  reverse_proxy 127.0.0.1:8170
}

ntfy.jgaga.tk {
  encode zstd gzip
  reverse_proxy 127.0.0.1:8080
}

synapse-admin.jgaga.tk {
  encode zstd gzip
  header Cache-Control "no-store"
  reverse_proxy 127.0.0.1:8100
}
EOF

cd /root/data/docker_data/caddy

  "restrictBaseUrl": "https://matrix.jgaga.tk"
}
EOF

docker-compose.yaml：

cat > /root/data/docker_data/synapse-admin/docker-compose.yaml <<'EOF'
services:
  synapse-admin:
    image: awesometechnologies/synapse-admin:latest
    container_name: synapse-admin
    restart: unless-stopped
    ports:
      - "127.0.0.1:8100:80"

      - ./config.json:/app/config.json:ro

  1. 打开 https://element.jgaga.tk。

  3. 用 @admin2:jgaga.tk 登录。
  4. 打开 https://synapse-admin.jgaga.tk，用同一个管理员账号登录，用户列表能显示。

目录和密钥不要乱放

从零安装完成后，只认这一套运行目录：

/root/data/docker_data/

数据库、媒体、signing key、token、密码都留在这套 root 管理目录里：

/root/data/docker_data/synapse/postgres/
/root/data/docker_data/synapse/data/media_store/
/root/data/docker_data/synapse/data/*.signing.key
/root/data/docker_data/_secrets/

这些内容不要复制到公开仓库、网页文章、聊天记录或普通用户目录。以后备份时可以备份它们，但要当作生产机密数据处理。
