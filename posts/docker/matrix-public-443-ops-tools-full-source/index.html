机密：Matrix 运维面板、MusicBot 和生产 Bot 小白部署手册

在已有 Matrix public-443 主栈上，从零部署 matrix-admin-tools 运维面板、Element Call MusicBot，以及生产正在使用的 matrix-bot-sync、matrix-bot-weather、matrix-bot-monitor 三个 OpenRC Bot 服务。

这篇只接着主栈文章往后做三件事：

  • matrix-admin-tools：网页运维面板，用来发 Server Notice 和管理媒体清理规则。
  • matrix-musicbot：进入 Element Call 播放音乐、直播流或普通音频。
  • matrix-bot-sync：接受邀请、处理“帮助/天气/设置天气时间/设置城市”等互动命令。
  • matrix-bot-weather：按用户配置的时间主动发送每日天气。
  • matrix-bot-monitor：检查磁盘、Docker、TURN、LiveKit、Element Call，并把告警发到指定用户。

本文假设主栈已经按上一篇手册完成：Synapse 本机监听 127.0.0.1:8008，Caddy 已接公网 80/443，管理员账号是 @admin2:jgaga.tk，管理员 token 已保存到：

/root/data/docker_data/_secrets/synapse-admin/access_token

下面所有命令默认已经 sudo -i 到 root shell。源码都在 GitHub 上，下一节给出两个仓库地址。

把本文里的 jgaga.tk、matrix.jgaga.tk、matrix-tools.jgaga.tk、192.227.236.209 换成自己的值即可。

获取源码

所有组件源码托管在 GitHub：

matrix-manager-ui（运维面板 + Bot 脚本 + OpenRC 服务文件）
https://github.com/hanigege/matrix-manager-ui

matrix-music-bot（音乐机器人）
https://github.com/hanigege/matrix-music-bot

建议直接在服务器上克隆：

ssh root@你的服务器IP
apt install -y git   # Debian/Ubuntu；Alpine 用 apk add git

# 运维面板源码（含 Bot 脚本）
install -d -m 755 /root/data/docker_data
git clone https://github.com/hanigege/matrix-manager-ui.git /root/data/docker_data/_ops-source

# 音乐机器人
git clone https://github.com/hanigege/matrix-music-bot.git /root/data/docker_data/_ops-musicbot

之后每个服务从对应的克隆目录拷贝模板。

准备账号和 token

三个组件需要两类 token：

matrix-admin-tools 读取管理员 token:
/root/data/docker_data/_secrets/synapse-admin/access_token

musicbot 和生产 Bot 使用普通 bot token:
/root/data/docker_data/_secrets/matrix-musicbot/access_token
/root/data/docker_data/_secrets/matrix-bot/access_token

先创建密钥目录：

install -d -m 700 \
  /root/data/docker_data/_secrets/matrix-musicbot \
  /root/data/docker_data/_secrets/matrix-bot

创建 musicbot 密码并注册账号：

openssl rand -base64 24 > /root/data/docker_data/_secrets/matrix-musicbot/musicbot-password
chmod 600 /root/data/docker_data/_secrets/matrix-musicbot/musicbot-password

musicbot_pass="$(cat /root/data/docker_data/_secrets/matrix-musicbot/musicbot-password)"
docker exec -it matrix-synapse register_new_matrix_user \
  -c /data/homeserver.yaml \
  -u musicbot \
  -p "$musicbot_pass" \
  --no-admin \
  http://127.0.0.1:8008

登录拿 musicbot token：

musicbot_pass="$(cat /root/data/docker_data/_secrets/matrix-musicbot/musicbot-password)"
curl -fsS -X POST 'http://127.0.0.1:8008/_matrix/client/v3/login' \
  -H 'Content-Type: application/json' \
  -d "$(jq -nc --arg user 'musicbot' --arg pass "$musicbot_pass" '{
    type:"m.login.password",
    identifier:{type:"m.id.user", user:$user},
    password:$pass,
    initial_device_display_name:"matrix-musicbot"
  }')" >/tmp/musicbot-login.json

jq -r '.access_token' /tmp/musicbot-login.json > /root/data/docker_data/_secrets/matrix-musicbot/access_token
chmod 600 /root/data/docker_data/_secrets/matrix-musicbot/access_token
rm -f /tmp/musicbot-login.json

创建生产 Bot 密码、注册账号并拿 token：

openssl rand -base64 24 > /root/data/docker_data/_secrets/matrix-bot/bot-password
chmod 600 /root/data/docker_data/_secrets/matrix-bot/bot-password

bot_pass="$(cat /root/data/docker_data/_secrets/matrix-bot/bot-password)"
docker exec -it matrix-synapse register_new_matrix_user \
  -c /data/homeserver.yaml \
  -u bot \
  -p "$bot_pass" \
  --no-admin \
  http://127.0.0.1:8008

curl -fsS -X POST 'http://127.0.0.1:8008/_matrix/client/v3/login' \
  -H 'Content-Type: application/json' \
  -d "$(jq -nc --arg user 'bot' --arg pass "$bot_pass" '{
    type:"m.login.password",
    identifier:{type:"m.id.user", user:$user},
    password:$pass,
    initial_device_display_name:"matrix-bot"
  }')" >/tmp/bot-login.json

jq -r '.access_token' /tmp/bot-login.json > /root/data/docker_data/_secrets/matrix-bot/access_token
chmod 600 /root/data/docker_data/_secrets/matrix-bot/access_token
rm -f /tmp/bot-login.json

验证 token：

for f in \
  /root/data/docker_data/_secrets/matrix-musicbot/access_token \
  /root/data/docker_data/_secrets/matrix-bot/access_token
 do
  TOKEN="$(cat "$f")"
  curl -fsS -H "Authorization: Bearer $TOKEN" \
    http://127.0.0.1:8008/_matrix/client/v3/account/whoami
  echo
 done

部署 matrix-admin-tools

创建目录并复制源码：

install -d -m 755 /root/data/docker_data/matrix-admin-tools
rsync -av /root/data/docker_data/_ops-source/matrix-admin-tools/ /root/data/docker_data/matrix-admin-tools/
install -d -m 755 \
  /root/data/docker_data/matrix-admin-tools/data \
  /root/data/docker_data/matrix-admin-tools/logs

确认 compose 里的 server name。本文模板默认 jgaga.tk，如果你的 server name 不同：

cd /root/data/docker_data/matrix-admin-tools
SERVER_NAME=你的server_name docker compose config >/tmp/matrix-admin-tools-compose-check.yaml

启动：

cd /root/data/docker_data/matrix-admin-tools
SERVER_NAME=jgaga.tk docker compose up -d --build

检查端口已监听：

curl -fsS http://127.0.0.1:8190 | grep 'Matrix 运维面板'
ss -lntp | grep ':8190'

给 Caddy 加公网入口。在 /root/data/docker_data/caddy/Caddyfile 添加：

matrix-tools.jgaga.tk {
  encode zstd gzip
  header Cache-Control "no-store"
  reverse_proxy 127.0.0.1:8190
}

重建 Caddy：

cd /root/data/docker_data/caddy
docker compose up -d --force-recreate

面板自带了 HTTP Basic Auth（用户名 admin，密码存在 _secrets/synapse-admin/admin-password）。浏览器打开 https://matrix-tools.jgaga.tk 后应弹出 Basic Auth 登录框，输入认证后能看到"当前自动清理规则"。

通知页面直达入口：

curl -I https://matrix-tools.jgaga.tk/notice

登录后访问 https://matrix-tools.jgaga.tk/notice 应显示同一个运维面板，并定位到 Server Notice 表单。GET /notice 只负责打开表单，不会发送通知；真正发信仍然必须提交 POST /notice，这样直达链接不会误触发全站通知。

部署 matrix-musicbot

创建目录并复制模板：

install -d -m 755 \
  /root/data/docker_data/matrix-musicbot/config \
  /root/data/docker_data/matrix-musicbot/logs \
  /root/data/docker_data/matrix-musicbot/data \
  /root/data/docker_data/matrix-musicbot/cache

rsync -av /root/data/docker_data/_ops-musicbot/docker-compose.yaml \
  /root/data/docker_data/matrix-musicbot/docker-compose.yaml
rsync -av /root/data/docker_data/_ops-musicbot/config/config.toml.example \
  /root/data/docker_data/matrix-musicbot/config/config.toml

把 token 写进配置：

MUSICBOT_TOKEN="$musicbot_token" python3 - <<'PY'
from pathlib import Path
import os
p = Path('/root/data/docker_data/matrix-musicbot/config/config.toml')
s = p.read_text()
s = s.replace('replace-with-musicbot-access-token', os.environ['MUSICBOT_TOKEN'])
p.write_text(s)
PY

设置权限。这个镜像内进程使用 UID 10001 写日志和缓存：

chown -R 10001:10001 \
  /root/data/docker_data/matrix-musicbot/logs \
  /root/data/docker_data/matrix-musicbot/data \
  /root/data/docker_data/matrix-musicbot/cache
chmod 644 /root/data/docker_data/matrix-musicbot/config/config.toml

启动：

cd /root/data/docker_data/matrix-musicbot
docker compose up -d
docker logs --tail=120 matrix-musicbot

日志里应看到类似 Startup checks passed、Bot ready。使用方式：

  1. 在 Element 里邀请 @musicbot:jgaga.tk 进房间。
  2. 进入房间右上角通话，自己先点“加入”。
  3. 在房间发 !join 让 bot 加入通话。
  4. 发 !play 音频URL 或普通可播放 URL。
  5. 发 !stop 停止，!leave 离开通话。

注意：!join 是让机器人进通话，不是让你的客户端进通话。你自己仍然要点 Element 的“加入”。

部署 Matrix 生产 Bot 栈

当前 public-443 栈把普通 Bot 拆成三个 OpenRC 服务：

matrix-bot-sync     接受邀请和处理用户命令
matrix-bot-weather  定时发送天气
matrix-bot-monitor  监控磁盘、容器、TURN、LiveKit、Element Call

三者共用同一个 @bot:jgaga.tk token：

/root/data/docker_data/_secrets/matrix-bot/access_token

复制脚本和配置模板：

install -d -m 755 /root/data/docker_data/synapse/scripts
rsync -av /root/data/docker_data/_ops-source/matrix-bot/ \
  /root/data/docker_data/synapse/scripts/
chmod 755 /root/data/docker_data/synapse/scripts/*.py \
  /root/data/docker_data/synapse/scripts/*.sh

如果你的 server name、默认收件人、城市或检查项不同，先改这两个配置：

/root/data/docker_data/synapse/scripts/matrix-bot-weather.json
/root/data/docker_data/synapse/scripts/matrix-bot-monitor.json

当前生产天气配置的关键形状是：

{
  "target_users": ["@honey:jgaga.tk", "@black:jgaga.tk", "@admin2:jgaga.tk", "@admin:jgaga.tk"],
  "default_weather_time": "05:00",
  "schedule_grace_minutes": 15,
  "weather_users": {
    "@honey:jgaga.tk": {"enabled": true, "time": "05:00"},
    "@black:jgaga.tk": {"enabled": true, "time": "05:00"},
    "@admin2:jgaga.tk": {"enabled": true, "time": "05:00"},
    "@admin:jgaga.tk": {"enabled": true, "time": "05:00"}
  },
  "direct_rooms": {},
  "sync_state_path": "/root/data/docker_data/synapse/scripts/matrix-bot-sync.state.json",
  "state_path": "/root/data/docker_data/synapse/scripts/matrix-bot-weather.state.json"
}

天气 Bot 没有 E2EE 密钥管理能力。脚本发现缓存私聊是加密房间时，会离开旧房间并重建不加密私聊；不要把它改成复用加密房间，否则定时推送会发不出去。

监控配置至少要覆盖当前生产容器名。旧文档或旧脚本里的 web-call 已经不是当前容器，当前是 element-call：

{
  "target_users": ["@black:jgaga.tk", "@honey:jgaga.tk", "@admin2:jgaga.tk"],
  "docker": {
    "expected_containers": [
      "matrix-admin-tools",
      "caddy",
      "element-web",
      "synapse-admin",
      "matrix-musicbot",
      "matrix-synapse",
      "postgres16",
      "ntfy",
      "element-call",
      "matrix-livekit-jwt-service",
      "livekit-server-test",
      "coturn"
    ]
  }
}

安装 OpenRC 服务：

rsync -av /root/data/docker_data/_ops-source/openrc/matrix-bot-sync /etc/init.d/matrix-bot-sync
rsync -av /root/data/docker_data/_ops-source/openrc/matrix-bot-weather /etc/init.d/matrix-bot-weather
rsync -av /root/data/docker_data/_ops-source/openrc/matrix-bot-monitor /etc/init.d/matrix-bot-monitor
chmod 755 /etc/init.d/matrix-bot-sync /etc/init.d/matrix-bot-weather /etc/init.d/matrix-bot-monitor

启动并加入默认运行级别：

rc-service matrix-bot-sync start
rc-service matrix-bot-weather start
rc-service matrix-bot-monitor start
rc-update add matrix-bot-sync default
rc-update add matrix-bot-weather default
rc-update add matrix-bot-monitor default
rc-status default | grep matrix-bot

交互验证：

  1. 在 Element 里邀请 @bot:jgaga.tk。
  2. 发送 帮助，应看到天气命令菜单。
  3. 发送 天气，应立即返回天气预报。
  4. 发送 设置天气时间 07:30，应保存你的每日推送时间。
  5. 看同步循环日志：
tail -80 /var/log/matrix-bot-sync-loop.log
tail -80 /var/log/matrix-bot-weather-loop.log
tail -80 /var/log/matrix-bot-monitor-loop.log

监控验证：

/root/data/docker_data/synapse/scripts/matrix-bot-monitor.py --test-alert
python3 - <<'PY'
import json
p='/root/data/docker_data/synapse/scripts/matrix-bot-monitor.state.json'
d=json.load(open(p))
print(json.dumps(d.get('direct_rooms', {}), ensure_ascii=False, indent=2))
PY

direct_rooms 里应有每个 target_users 对应的私聊房间。只看 rc-service ... status 不够，监控脚本必须真的能给目标用户发出测试告警。

synapse-media-purge 的 OpenRC 文件也随源码保留，但不要默认启用。它会触发生产媒体清理，确认策略、备份和 keep_profiles=true 语义以后再执行：

# 媒体清理会删除命中规则的生产媒体；确认策略和大备份后再启用自动服务。
rsync -av /root/data/docker_data/_ops-source/openrc/synapse-media-purge /etc/init.d/synapse-media-purge
chmod 755 /etc/init.d/synapse-media-purge
# rc-update add synapse-media-purge default
# rc-service synapse-media-purge start

最终验收

容器：

docker ps --format '{{.Names}} {{.Status}}' | sort | grep -E 'matrix-admin-tools|matrix-musicbot'

OpenRC：

rc-status default | grep -E 'matrix-bot-sync|matrix-bot-weather|matrix-bot-monitor'

运维面板：

curl -fsS http://127.0.0.1:8190 | grep 'Matrix 运维面板'
curl -fsS http://127.0.0.1:8190/notice | grep 'Server Notice'
curl -I https://matrix-tools.jgaga.tk
curl -I https://matrix-tools.jgaga.tk/notice

公网 matrix-tools.jgaga.tk 未带密码时返回 401 是正确的，这是面板自带的 Basic Auth 保护。用浏览器输入 admin 密码后应能打开页面。不要把 GET /notice 的历史 404 当成 DNS 或证书问题，它只是后端没给直达通知页配置 GET 路由。

MusicBot：

docker logs --tail=120 matrix-musicbot

生产 Bot：

tail -80 /var/log/matrix-bot-sync-loop.log
tail -80 /var/log/matrix-bot-weather-loop.log
tail -80 /var/log/matrix-bot-monitor-loop.log
/root/data/docker_data/synapse/scripts/matrix-bot-monitor.py --test-alert

如果这些都正常，运维面板、MusicBot 和三段式生产 Bot 栈就部署完成。真实 token 和密码只存在 /root/data/docker_data/_secrets/，不要写进文章、公开仓库或聊天记录。
