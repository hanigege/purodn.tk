RouterOS 旁路由环境下配置英雄联盟美服

在 RouterOS 主路由和 sing-box 旁路由环境中，让英雄联盟美服 Riot 流量稳定直连，避免 fake-ip、透明代理和 invalid 防火墙规则造成登录、更新或游戏断连。

这篇记录一套已经实战跑通的配置：主路由是 RouterOS，旁路由是运行 sing-box 的 Linux 主机，客户端把网关指向旁路由。目标是让英雄联盟美服和 Riot 客户端稳定运行，不被 fake-ip、代理规则、RouterOS 的 Drop invalid 或非对称路由误伤。

这不是“给游戏强行走代理”的方案。美服 LOL 更适合稳定直连：Riot 登录、启动器、更新和游戏内 UDP 流量都要尽量拿到真实解析结果，并从旁路由里明确放行直连。

1. 网络拓扑

示例网络如下，按自己的环境替换地址：

角色 示例地址 说明
RouterOS 主路由 10.20.20.1 负责拨号、NAT、防火墙、FastTrack
sing-box 旁路由 10.20.20.6 Linux 主机，客户端网关指向它
游戏电脑 10.20.20.100 安装 Riot Client 和 League of Legends

典型流量路径是：

游戏电脑 -> sing-box 旁路由 -> RouterOS -> Internet

但是旁路由网络很容易出现非对称回程：

去程：游戏电脑 -> 旁路由 -> RouterOS -> Internet
回程：Internet -> RouterOS -> 游戏电脑

这对 TCP 网页访问通常不明显，但对 UDP 游戏流量和 Riot 客户端的长连接很敏感。如果 RouterOS 在 forward 链里严格丢弃 invalid，就可能把正常回程包当成异常包扔掉，表现为登录卡住、更新慢、进游戏掉线或延迟异常。

2. 先确认客户端网关

游戏电脑必须确实经过旁路由。Windows 可以查看：

ipconfig

重点看默认网关：

Default Gateway . . . . . . . . . : 10.20.20.6

如果游戏电脑默认网关还是 RouterOS，例如 10.20.20.1，那本文的 sing-box 分流规则不会接管它。

3. RouterOS 防火墙处理

3.1 检查 invalid 规则

在 RouterOS 终端执行：

/ip firewall filter print where chain=forward connection-state=invalid
/ipv6 firewall filter print where chain=forward connection-state=invalid

如果看到类似规则：

chain=forward action=drop connection-state=invalid

并且你的网络里有旁路由透明代理，就要谨慎。它可能误杀 LOL、Telegram 或其他 UDP 应用。

3.2 推荐做法：停用 forward invalid drop

先停用，不要急着删除：

/ip firewall filter disable [find chain=forward connection-state=invalid]
/ipv6 firewall filter disable [find chain=forward connection-state=invalid]

停用后立刻测试 Riot Client 和 LOL。如果登录、更新、进游戏恢复正常，再决定是否删除。

3.3 确认内网转发规则仍然存在

停用 invalid drop 后，forward 链仍应有允许 LAN 出站的规则。例如：

/ip firewall filter add chain=forward action=accept in-interface=bridge1 comment="Allow LAN forward"

如果你使用 interface list，也可以写成：

/interface list add name=LAN
/interface list member add list=LAN interface=bridge1
/ip firewall filter add chain=forward action=accept in-interface-list=LAN comment="Allow LAN forward"

这条规则的目的不是放弃安全，而是让内网主动发起的流量正常出站。真正的入站暴露仍然应该由单独的 WAN 入站规则控制。

4. sing-box 规则文件准备

本文使用 geosite-riot 规则集。你需要确认 sing-box 机器上有 Riot 的规则文件：

ls -lh /etc/sing-box/rules/geosite/riot.srs

如果不存在，需要先更新或生成规则集。不同维护脚本路径不一样，原则是让下面这个文件存在：

/etc/sing-box/rules/geosite/riot.srs

5. DNS 配置

Riot/LOL 不建议走 fake-ip。Riot 客户端和游戏连接对 UDP、登录域名和地区服务比较敏感，拿到 fake-ip 后再由透明代理还原，容易在某些阶段出问题。

编辑 sing-box DNS 配置：

nano /etc/sing-box/conf/02_dns.json

在 dns.servers 中保留一个远程 DNS，例如：

{
  "tag": "remote-dns",
  "type": "https",
  "server": "1.1.1.1",
  "server_port": 443,
  "path": "/dns-query",
  "tls": {
    "server_name": "cloudflare-dns.com"
  },
  "detour": "Proxy",
  "reuse": true,
  "pipeline": true
}

然后在 dns.rules 靠前位置加入 Riot 规则，让 Riot 域名用真实 DNS 解析：

{
  "rule_set": "geosite-riot",
  "action": "route",
  "server": "remote-dns",
  "rewrite_ttl": 60
}

上下文示例：

{
  "dns": {
    "final": "remote-dns",
    "servers": [
      {
        "tag": "remote-dns",
        "type": "https",
        "server": "1.1.1.1",
        "server_port": 443,
        "path": "/dns-query",
        "tls": {
          "server_name": "cloudflare-dns.com"
        },
        "detour": "Proxy",
        "reuse": true,
        "pipeline": true
      },
      {
        "tag": "fakeip-dns",
        "type": "fakeip",
        "inet4_range": "28.0.0.0/8",
        "inet6_range": "2001:2::/64"
      }
    ],
    "rules": [
      {
        "inbound": "fake-in",
        "action": "route",
        "rewrite_ttl": 60,
        "disable_cache": false,
        "server": "fakeip-dns"
      },
      {
        "rule_set": "geosite-riot",
        "action": "route",
        "server": "remote-dns",
        "rewrite_ttl": 60
      }
    ]
  }
}

重点是：geosite-riot 要放在大范围 fake-ip 规则之前。

6. route 配置

编辑路由配置：

nano /etc/sing-box/conf/05_route.json

在 route.rule_set 里加入 Riot 规则文件：

{
  "type": "local",
  "format": "binary",
  "tag": "geosite-riot",
  "path": "/etc/sing-box/rules/geosite/riot.srs"
}

上下文示例：

{
  "route": {
    "default_domain_resolver": "remote-dns",
    "auto_detect_interface": true,
    "final": "Proxy",
    "rule_set": [
      {
        "type": "local",
        "format": "binary",
        "tag": "geosite-youtube",
        "path": "/etc/sing-box/rules/geosite/youtube.srs"
      },
      {
        "type": "local",
        "format": "binary",
        "tag": "geosite-riot",
        "path": "/etc/sing-box/rules/geosite/riot.srs"
      }
    ],
    "rules": []
  }
}

然后在 route.rules 靠前位置加入 Riot 直连规则：

{
  "rule_set": "geosite-riot",
  "outbound": "direct"
}

上下文示例：

{
  "route": {
    "rules": [
      {
        "inbound": "dns-in",
        "action": "hijack-dns"
      },
      {
        "inbound": "fake-in",
        "action": "hijack-dns"
      },
      {
        "rule_set": "geosite-riot",
        "outbound": "direct"
      },
      {
        "ip_is_private": true,
        "outbound": "direct"
      }
    ],
    "final": "Proxy"
  }
}

这里的顺序很重要。Riot 直连规则应该放在通用国外代理规则之前，也应该放在最后兜底 final: Proxy 之前。

7. 不要把 Riot 放进代理选择器

很多人会想给 Riot 单独建一个 LOL 或 Riot 选择器，然后选代理节点。这个方案不适合本文目标。

建议保持：

{
  "rule_set": "geosite-riot",
  "outbound": "direct"
}

原因是 LOL 游戏流量里有大量 UDP 和地区服务，强行代理不一定更低延迟，反而容易因为节点 UDP、NAT、MTU、fake-ip 还原和透明代理链路导致抖动。

8. 检查配置并重启

先检查配置：

sing-box check -C /etc/sing-box/conf/

没有报错再重启：

systemctl restart sing-box
systemctl status sing-box --no-pager

如果使用 OpenRC 或其他服务管理器，按自己的系统替换重启命令。

9. 验证 Riot 解析没有进入 fake-ip

在旁路由上测试 Riot 域名解析。示例域名可以用：

dig riotgames.com
dig auth.riotgames.com

如果你的系统没有 dig：

getent hosts riotgames.com
getent hosts auth.riotgames.com

正常结果应该是真实公网 IP，不应该是 fake-ip 网段。例如本文配置里的 fake-ip 是：

28.0.0.0/8

如果解析结果长这样，就是不对：

28.x.x.x

应该回去检查 02_dns.json 里的 geosite-riot DNS 规则是否放在 fake-ip 大规则前面。

10. 验证路由命中直连

查看 sing-box 日志：

journalctl -u sing-box -f

然后打开 Riot Client 登录美服账号，再启动英雄联盟。你应该看到 Riot 相关域名命中 direct，而不是进入 Proxy。

也可以用面板 API 看当前选择器状态：

curl -s http://127.0.0.1:9090/proxies | head

这里不是为了把 Riot 切到某个节点，而是确认整体代理服务正常。

11. 验证 RouterOS 没有继续误杀

在 RouterOS 上看 invalid 规则是否还在启用：

/ip firewall filter print where chain=forward connection-state=invalid
/ipv6 firewall filter print where chain=forward connection-state=invalid

如果它仍然启用，并且 LOL 进游戏或语音出现异常，先停用：

/ip firewall filter disable [find chain=forward connection-state=invalid]
/ipv6 firewall filter disable [find chain=forward connection-state=invalid]

然后重新登录 Riot Client、进入训练模式测试 10 分钟。

12. 常见现象

Riot Client 能打开，但登录慢

优先检查 DNS。Riot 域名不要进 fake-ip，geosite-riot DNS 规则要靠前。

能登录，但进游戏掉线

优先检查 RouterOS 的 forward invalid drop。旁路由非对称回程时，这条规则很容易误杀 UDP。

其他网站都正常，只有 LOL 不正常

这通常不是节点问题，而是游戏流量不应该走通用代理链路。先确认 geosite-riot -> direct。

禁用 invalid drop 后担心不安全

安全边界应该放在 WAN 入站和端口暴露规则上。内网主动发起的 forward 流量，需要给旁路由网络留下容错空间。你仍然可以保留 WAN 入站 drop、只放行必要端口、关闭不需要的公网服务。

13. 最终检查清单

完成后逐项确认：

  • 游戏电脑默认网关是旁路由。
  • riot.srs 文件存在。
  • DNS 规则里 geosite-riot 不走 fake-ip。
  • route 规则里 geosite-riot 走 direct。
  • RouterOS 没有启用会误杀旁路由 UDP 的 forward invalid drop。
  • sing-box check -C /etc/sing-box/conf/ 通过。
  • Riot Client 能登录。
  • LOL 美服能进训练模式并稳定 10 分钟以上。

这套配置的核心不是复杂，而是把 Riot 从“通用代理逻辑”里摘出来：真实 DNS、明确直连、RouterOS 不误杀。做到这三点，美服 LOL 在旁路由环境里就会稳定很多。
