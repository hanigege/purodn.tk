FreeBSD 15 下配置 Hysteria2 服务器端

在 FreeBSD 15 上安装最新版 Hysteria2，配置 TLS、自签证书、HTTP/3 伪装、可选 Salamander 混淆、rc.d 服务、升级流程和 sing-box 客户端。

本文记录在 FreeBSD 15 上部署 Hysteria2 服务端的完整流程。Hysteria2 基于 QUIC，默认会伪装成标准 HTTP/3 流量；如果网络环境会针对 QUIC 或 HTTP/3 做阻断，可以再启用 sing-box 兼容的 salamander 混淆。

截至 2026-05-24，Hysteria2 最新稳定版本是 app/v2.9.2。这个版本包含安全修复，建议新部署和旧服务都升级到这一版。

1. 部署思路

最稳妥的服务端配置思路是：

  • 监听 UDP 443，这也是官方默认 HTTP/3 端口。
  • 使用 TLS 证书；如果没有正式域名，可以先用自签证书。
  • 使用 masquerade.proxy 伪装成正常网站。
  • 个人自用时，服务端不写 bandwidth 和 ignoreClientBandwidth，把带宽参数交给客户端。
  • 不手动修改 QUIC 窗口，官方默认已经是 8 MB stream 和 20 MB connection。
  • 如果网络环境针对 HTTP/3 或 QUIC 阻断，再开启 sing-box 兼容的 obfs.type: salamander。

官方文档里说明，listen 省略时会默认监听 :443，因为这是默认 HTTP/3 端口。这里为了文章清晰，仍然显式写出 listen: :443。

2. 安装 Hysteria2 最新版

建议用 root 用户操作。监听 443 这类 1024 以下端口也需要 root 权限，或额外配置权限。

设置版本变量：

HY2_VERSION='app/v2.9.2'

下载 FreeBSD amd64 官方二进制：

fetch -o /tmp/hysteria "https://github.com/apernet/hysteria/releases/download/${HY2_VERSION}/hysteria-freebsd-amd64"
install -m 755 /tmp/hysteria /usr/local/bin/hysteria
rm -f /tmp/hysteria

确认版本：

hysteria version

如果后续有新版，只需要改 HY2_VERSION，重新执行下载和 install 命令即可。

3. 生成自签 TLS 证书

如果你有正式域名和证书，建议直接使用正式证书。没有正式证书时，可以先用自签证书跑起来。

创建配置目录：

mkdir -p /usr/local/etc/hysteria

生成 ECDSA 自签证书。下面示例使用 www.cloudflarestatus.com 作为伪装 SNI。它是一个正常 HTTPS 站点，适合用作示例里的伪装域名。

openssl req -x509 -nodes -newkey ec -pkeyopt ec_paramgen_curve:prime256v1 \
  -keyout /usr/local/etc/hysteria/server.key \
  -out /usr/local/etc/hysteria/server.crt \
  -subj "/CN=www.cloudflarestatus.com" \
  -days 36500

设置权限：

chmod 644 /usr/local/etc/hysteria/server.crt
chmod 600 /usr/local/etc/hysteria/server.key

配置文件里会引用这两个路径：

tls:
  cert: /usr/local/etc/hysteria/server.crt
  key: /usr/local/etc/hysteria/server.key

使用自签证书时，客户端需要开启 insecure: true，或者导入对应证书后再做严格校验。

自签证书的 CN 和客户端里的 server_name 最好保持一致；配置里的 masquerade.proxy.url 也建议使用同一个域名。严格来说，masquerade 是认证失败时返回的伪装网站，不等同于证书校验，但三者统一最不容易给自己挖坑。

4. 写入最靠谱的服务端配置

创建配置文件：

nvim /usr/local/etc/hysteria/config.yaml

写入下面这份基础配置。先把 password 改成自己的强密码：

listen: :443

tls:
  cert: /usr/local/etc/hysteria/server.crt
  key: /usr/local/etc/hysteria/server.key

auth:
  type: password
  password: "请改成一个足够长的强密码"

masquerade:
  type: proxy
  proxy:
    url: https://www.cloudflarestatus.com/
    rewriteHost: true

log:
  level: warn

这份配置的特点是简单、可靠、接近官方推荐默认值：

  • listen: :443：监听标准 HTTP/3 UDP 端口。
  • auth.password：最简单的单用户密码认证。
  • masquerade.proxy：伪装成正常 HTTP/3 网站请求。
  • 不写 bandwidth：个人自用时，把带宽参数交给客户端更灵活。
  • 不写 quic：保留官方默认 QUIC 参数，避免过度调参导致不稳定。

5. 可选：启用 Salamander 混淆

你提到的“h3”可以理解为两件事：

  • Hysteria2 默认就会模仿 HTTP/3，这是基础形态。
  • 如果网络专门阻断 QUIC 或 HTTP/3，可以额外开启混淆。因为你的客户端是 sing-box，所以这里选 sing-box 兼容的 salamander。

注意：启用混淆后，服务端就不再是一个标准 HTTP/3 服务器，客户端也必须配置完全相同的 obfs 密码，否则连不上。

如果你确认需要混淆，可以在 config.yaml 里加入：

obfs:
  type: salamander
  salamander:
    password: "请改成另一个足够长的混淆密码"

最终配置示例：

listen: :443

tls:
  cert: /usr/local/etc/hysteria/server.crt
  key: /usr/local/etc/hysteria/server.key

auth:
  type: password
  password: "请改成一个足够长的强密码"

obfs:
  type: salamander
  salamander:
    password: "请改成另一个足够长的混淆密码"

masquerade:
  type: proxy
  proxy:
    url: https://www.cloudflarestatus.com/
    rewriteHost: true

log:
  level: warn

如果只是普通线路，先不要开 obfs。默认 HTTP/3 伪装更像正常 H3 服务，也更容易排查问题。

6. 创建 rc.d 服务

创建 FreeBSD 服务脚本：

nvim /usr/local/etc/rc.d/hysteria

写入：

#!/bin/sh

# PROVIDE: hysteria
# REQUIRE: NETWORKING
# KEYWORD: shutdown

. /etc/rc.subr

name="hysteria"
rcvar="hysteria_enable"

load_rc_config $name

: ${hysteria_enable:="NO"}
: ${hysteria_config:="/usr/local/etc/hysteria/config.yaml"}
: ${hysteria_log:="/var/log/hysteria.log"}

pidfile="/var/run/${name}.pid"
procname="/usr/local/bin/hysteria"

command="/usr/sbin/daemon"
command_args="-r -P ${pidfile} -o ${hysteria_log} ${procname} server -c ${hysteria_config}"

run_rc_command "$1"

赋予执行权限并创建日志文件：

chmod +x /usr/local/etc/rc.d/hysteria
touch /var/log/hysteria.log
chmod 644 /var/log/hysteria.log

写入 /etc/rc.conf：

sysrc hysteria_enable="YES"
sysrc hysteria_config="/usr/local/etc/hysteria/config.yaml"

7. 启动和检查

先检查配置是否能被 Hysteria2 正常读取：

hysteria server -c /usr/local/etc/hysteria/config.yaml

如果前台运行没有报错，按 Ctrl+C 退出，然后启动后台服务：

service hysteria start

检查状态：

service hysteria status

查看日志：

tail -f /var/log/hysteria.log

修改配置后重启：

service hysteria restart

确认 UDP 443 已监听：

sockstat -4 -6 -l | grep hysteria

8. FreeBSD 防火墙放行

Hysteria2 主要使用 UDP。如果服务器启用了防火墙，需要放行 UDP 443。

如果使用云服务器安全组，也要在控制台放行 UDP 443，不要只放行 TCP 443。

本文前面的服务端和 sing-box 客户端示例都使用 443，所以这里也只放行 UDP 443，上下文保持一致。

9. FreeBSD UDP 缓冲优化

这一节建议安装后就做。Hysteria2 基于 QUIC/UDP，刚装好的 FreeBSD 可以正常连通，但默认 UDP 接收缓冲可能只有几十 KB。比如实机遇到过：

net.inet.udp.recvspace: 42080

这个值对网页、聊天这类轻量请求可能够用，但不适合作为高吞吐 HY2 服务端。跨国、高延迟链路下，YouTube 4K、测速、下载、多设备共享都可能因为 UDP socket buffer 太小而跑不满。先不要乱调 Hysteria2 的 QUIC 窗口，把 FreeBSD 系统层 UDP 缓冲准备好即可。

9.1 先查看当前值

sysctl kern.ipc.maxsockbuf net.inet.udp.recvspace net.inet.ip.intr_queue_maxlen

如果 net.inet.udp.recvspace 还是 42080 或类似几十 KB 的值，建议继续按下面步骤修改。

9.2 备份 sysctl.conf

cp -a /etc/sysctl.conf /etc/sysctl.conf.bak.hy2-$(date +%Y%m%d%H%M%S)

9.3 写入推荐参数

下面命令会先移除旧的同名配置，再写入新的 HY2 UDP 参数，避免重复追加：

grep -v -E '^(kern\.ipc\.maxsockbuf|net\.inet\.udp\.recvspace|net\.inet\.ip\.intr_queue_maxlen)=' \
  /etc/sysctl.conf > /tmp/sysctl.conf.hy2

cat >> /tmp/sysctl.conf.hy2 <<'EOF'

# Hysteria2 / QUIC UDP tuning
kern.ipc.maxsockbuf=33554432
net.inet.udp.recvspace=4194304
net.inet.ip.intr_queue_maxlen=4096
EOF

install -m 0644 /tmp/sysctl.conf.hy2 /etc/sysctl.conf
rm -f /tmp/sysctl.conf.hy2

参数含义：

  • kern.ipc.maxsockbuf=33554432：允许 socket buffer 放大到 32 MB。
  • net.inet.udp.recvspace=4194304：把 UDP 默认接收缓冲提高到 4 MB。
  • net.inet.ip.intr_queue_maxlen=4096：提高 IP 输入队列长度，减少突发流量下的队列压力。

9.4 立即生效

sysctl kern.ipc.maxsockbuf=33554432
sysctl net.inet.udp.recvspace=4194304
sysctl net.inet.ip.intr_queue_maxlen=4096

再次确认：

sysctl kern.ipc.maxsockbuf net.inet.udp.recvspace net.inet.ip.intr_queue_maxlen

应该看到类似：

kern.ipc.maxsockbuf: 33554432
net.inet.udp.recvspace: 4194304
net.inet.ip.intr_queue_maxlen: 4096

9.5 重启 Hysteria2

service hysteria restart

确认 UDP 443 仍在监听：

sockstat -4 -6 -l | grep hysteria

正常应看到类似：

root hysteria  12345  4  udp46  *:443  *:*

9.6 常见避坑

不要把下面这类 net.isr.* 参数直接写进 /etc/sysctl.conf：

net.isr.defaultqlimit
net.isr.maxqlimit

在 FreeBSD 15 上，它们属于启动期 tunable，运行时用 sysctl -f /etc/sysctl.conf 会报 read only tunable。普通 HY2 服务端先不用碰它们，前面的 UDP 缓冲参数已经能解决最常见的默认值过小问题。

如果重启时日志里出现 listen udp :443: bind: address already in use，说明旧进程或旧 daemon 监控进程还占着端口。先看进程：

ps auxww | egrep 'daemon: /usr/local/bin/hysteria|/usr/local/bin/hysteria server'
sockstat -4 -6 -l | grep ':443'

确认只有旧的 Hysteria2 占用端口后，再停掉旧进程并重新启动服务：

service hysteria stop
pkill -TERM -f '/usr/local/bin/hysteria server'
rm -f /var/run/hysteria.pid
service hysteria start

10. sing-box 客户端配置

下面给一份 sing-box 客户端 outbound 示例。把 你的服务器IP或域名、密码和带宽改成自己的即可。

如果服务端使用上面的自签证书，并且证书 CN 是 www.cloudflarestatus.com，客户端的 server_name 也写这个：

{
  "type": "hysteria2",
  "tag": "hy2-freebsd",
  "server": "你的服务器IP或域名",
  "server_port": 443,
  "up_mbps": 50,
  "down_mbps": 200,
  "password": "服务端 auth.password 里的同一个密码",
  "tls": {
    "enabled": true,
    "server_name": "www.cloudflarestatus.com",
    "insecure": true
  }
}

如果服务端没有开启 obfs，客户端也不要写 obfs。两边必须一致。

如果服务端开启了 Salamander 混淆，sing-box 客户端要加入同样的 obfs：

{
  "type": "hysteria2",
  "tag": "hy2-freebsd",
  "server": "你的服务器IP或域名",
  "server_port": 443,
  "up_mbps": 50,
  "down_mbps": 200,
  "password": "服务端 auth.password 里的同一个密码",
  "obfs": {
    "type": "salamander",
    "password": "服务端 obfs.salamander.password 里的同一个密码"
  },
  "tls": {
    "enabled": true,
    "server_name": "www.cloudflarestatus.com",
    "insecure": true
  }
}

up_mbps 和 down_mbps 是客户端告诉 Hysteria2 的可用带宽。家庭宽带、VPS 线路差异很大，不要照抄 1000。比如本地上行 50 Mbps、下行 200 Mbps，就按上面的例子写。

如果你以后换成正式域名和可信证书，tls.insecure 可以改成 false 或直接删除。

11. 升级 Hysteria2

后续想升级时，先打开 Hysteria2 官方 Releases 页面，确认最新 tag。例如当前版本是 app/v2.9.2，如果以后看到 app/v2.9.3 或更高版本，就把下面的 HY2_VERSION 改掉。

先备份当前二进制和配置：

cp /usr/local/bin/hysteria /usr/local/bin/hysteria.bak.$(date +%Y%m%d%H%M%S)
cp /usr/local/etc/hysteria/config.yaml /usr/local/etc/hysteria/config.yaml.bak.$(date +%Y%m%d%H%M%S)

停止服务：

service hysteria stop

下载并覆盖新版：

HY2_VERSION='app/v2.9.2'
fetch -o /tmp/hysteria "https://github.com/apernet/hysteria/releases/download/${HY2_VERSION}/hysteria-freebsd-amd64"
install -m 755 /tmp/hysteria /usr/local/bin/hysteria
rm -f /tmp/hysteria

检查版本和配置：

hysteria version
hysteria server -c /usr/local/etc/hysteria/config.yaml

如果前台运行没有报错，按 Ctrl+C 退出，再启动服务：

service hysteria start

确认服务状态、监听端口和日志：

service hysteria status
sockstat -4 -6 -l | grep hysteria
tail -f /var/log/hysteria.log

如果新版启动失败，可以临时回滚旧二进制：

service hysteria stop
cp /usr/local/bin/hysteria.bak.时间戳 /usr/local/bin/hysteria
service hysteria start

完成

到这里，FreeBSD 15 上的 Hysteria2 服务端就配置好了。

日常维护主要看四件事：

  • /usr/local/etc/hysteria/config.yaml
  • /var/log/hysteria.log
  • service hysteria status
  • 云服务器和 FreeBSD 防火墙是否放行 UDP 端口
