FreeBSD 普通用户下安装 Hysteria2 服务端

普通用户在 FreeBSD 上安装最新版 Hysteria2，使用 tmux 管理后台运行，通过 crontab 自启，并配置 HTTP/3 伪装和 sing-box 客户端。

这篇记录普通用户权限下部署 Hysteria2 服务端的方法。它不依赖 rc.d，而是用 tmux 保持后台运行，再通过用户自己的 crontab 做开机自启。

普通用户不能监听 1024 以下的特权端口，所以这里统一使用 UDP 8443。如果你有 root 权限并希望使用 UDP 443，建议看 root 版文章。

截至 2026-05-24，Hysteria2 最新稳定版本是 app/v2.9.2。这个版本包含安全修复，新部署建议直接使用这一版。

1. 安装 tmux

启动脚本依赖 tmux，需要先安装：

pkg install tmux

如果没有 root 权限，需要让管理员提前安装。

2. 安装 Hysteria2

创建普通用户自己的程序和配置目录：

mkdir -p ~/.usr/local/bin
mkdir -p ~/.usr/local/etc/hysteria

下载并安装 FreeBSD amd64 官方二进制：

HY2_VERSION='app/v2.9.2'

fetch -o /tmp/hysteria "https://github.com/apernet/hysteria/releases/download/${HY2_VERSION}/hysteria-freebsd-amd64"
install -m 755 /tmp/hysteria ~/.usr/local/bin/hysteria
rm -f /tmp/hysteria

确认可以运行：

~/.usr/local/bin/hysteria version

3. 生成自签证书

这里生成一套自签证书。示例使用 www.cloudflarestatus.com 作为伪装 SNI，它是一个正常 HTTPS 站点，适合用作示例里的伪装域名。

openssl req -x509 -nodes -newkey ec -pkeyopt ec_paramgen_curve:prime256v1 \
  -keyout ~/.usr/local/etc/hysteria/server.key \
  -out ~/.usr/local/etc/hysteria/server.crt \
  -subj "/CN=www.cloudflarestatus.com" \
  -days 36500

设置证书权限：

chmod 644 ~/.usr/local/etc/hysteria/server.crt
chmod 600 ~/.usr/local/etc/hysteria/server.key

自签证书的 CN、sing-box 客户端里的 server_name、服务端 masquerade.proxy.url 建议保持一致。严格来说，masquerade 是认证失败时返回的伪装网站，不等同于证书校验，但三者统一最不容易出错。

4. 创建 hctl 管理脚本

在普通用户终端执行下面整段代码，生成 ~/bin/hctl：

mkdir -p ~/bin

cat <<'EOF' > ~/bin/hctl
#!/bin/sh

HYSTERIA_BIN="$HOME/.usr/local/bin/hysteria"
CONFIG_FILE="$HOME/.usr/local/etc/hysteria/config.yaml"
LOG_FILE="$HOME/.usr/local/etc/hysteria/hysteria.log"
SESSION_NAME="hysteria"

if [ ! -x "$HYSTERIA_BIN" ]; then
    echo "error: executable not found: $HYSTERIA_BIN"
    exit 1
fi

case "$1" in
    start)
        if tmux has-session -t "$SESSION_NAME" 2>/dev/null; then
            echo "hysteria is already running in tmux session: $SESSION_NAME"
        else
            mkdir -p "$(dirname "$LOG_FILE")"
            tmux new-session -d -s "$SESSION_NAME" "$HYSTERIA_BIN server -c $CONFIG_FILE >> $LOG_FILE 2>&1"
            echo "hysteria started"
        fi
        ;;
    stop)
        if tmux has-session -t "$SESSION_NAME" 2>/dev/null; then
            tmux kill-session -t "$SESSION_NAME"
            echo "hysteria stopped"
        else
            echo "hysteria is not running"
        fi
        ;;
    restart)
        "$0" stop
        sleep 1
        "$0" start
        ;;
    status)
        if tmux has-session -t "$SESSION_NAME" 2>/dev/null; then
            echo "running"
        else
            echo "stopped"
        fi
        ;;
    log)
        if [ -f "$LOG_FILE" ]; then
            tail -f "$LOG_FILE"
        else
            echo "log file does not exist yet: $LOG_FILE"
        fi
        ;;
    attach)
        if tmux has-session -t "$SESSION_NAME" 2>/dev/null; then
            tmux attach-session -t "$SESSION_NAME"
        else
            echo "hysteria is not running"
        fi
        ;;
    *)
        echo "usage: hctl {start|stop|restart|status|log|attach}"
        exit 1
        ;;
esac
EOF

chmod +x ~/bin/hctl

脚本里全部使用 $HOME 拼接绝对路径，避免 cron 环境里找不到文件。

5. 写入服务端配置

编辑配置文件：

nvim ~/.usr/local/etc/hysteria/config.yaml

YAML 配置里不要写 ~，因为 Hysteria2 不会把它自动展开为用户主目录。证书路径必须写完整绝对路径。

先查看当前 home 目录：

echo "$HOME"

下面示例假设用户是 felix，home 是 /home/felix。如果你的用户名不是 felix，把路径改成自己的真实 home。

listen: :8443

tls:
  cert: /home/felix/.usr/local/etc/hysteria/server.crt
  key: /home/felix/.usr/local/etc/hysteria/server.key

auth:
  type: password
  password: "请改成一个足够长的强密码"

masquerade:
  type: proxy
  proxy:
    url: https://www.cloudflarestatus.com/
    rewriteHost: true

log:
  level: warn

如果你的 home 是 /home/mana，TLS 路径要改成：

tls:
  cert: /home/mana/.usr/local/etc/hysteria/server.crt
  key: /home/mana/.usr/local/etc/hysteria/server.key

这份配置和 root 版思路一致：默认 HTTP/3 伪装，不写 bandwidth，不手动改 QUIC 窗口。普通用户版唯一不同点是监听 UDP 8443，因为普通用户不能直接监听 UDP 443。

6. 可选：启用 Salamander 混淆

如果网络专门阻断 QUIC 或 HTTP/3，可以额外开启混淆。因为客户端是 sing-box，这里使用 sing-box 兼容的 salamander。

服务端添加：

obfs:
  type: salamander
  salamander:
    password: "请改成另一个足够长的混淆密码"

开启后，客户端也必须写同样的 obfs，否则连不上。如果普通线路能正常连接，先不要开 obfs，默认 HTTP/3 伪装更容易排查。

7. 前台测试

怀疑配置有问题时，先前台运行测试：

~/.usr/local/bin/hysteria server -c ~/.usr/local/etc/hysteria/config.yaml

检查是否有端口占用、权限错误或 YAML 语法错误。确认正常后按 Ctrl+C 停止，再交给脚本后台运行。

8. 日常控制命令

启动：

~/bin/hctl start

停止：

~/bin/hctl stop

重启：

~/bin/hctl restart

查看状态：

~/bin/hctl status

查看实时日志：

~/bin/hctl log

进入后台控制台：

~/bin/hctl attach

退出 tmux 并保持后台运行：

Ctrl+b
d

确认 UDP 8443 已监听：

sockstat -4 -6 -l | grep hysteria

9. 设置开机自启

编辑当前用户的 crontab：

crontab -e

在最后添加一行。路径要写成实际 home 目录，不要写 ~：

@reboot /home/felix/bin/hctl start

如果用户名不是 felix，先查看自己的 home 目录：

echo "$HOME"

例如输出是 /home/mana，就写：

@reboot /home/mana/bin/hctl start

启动链路就是：开机后 cron 执行 hctl start，然后 tmux 在后台保持 Hysteria2 运行。

10. 防火墙和安全组

本文统一使用 UDP 8443。如果服务器启用了 FreeBSD 防火墙，需要放行 UDP 8443。

如果是云服务器，还要在云服务商安全组里放行 UDP 8443，不要只放行 TCP 8443。

11. sing-box 客户端配置

下面是 sing-box 客户端 outbound 示例。把 你的服务器IP或域名、密码和带宽改成自己的即可。

如果服务端使用上面的自签证书，并且证书 CN 是 www.cloudflarestatus.com，客户端的 server_name 也写这个：

{
  "type": "hysteria2",
  "tag": "hy2-freebsd-user",
  "server": "你的服务器IP或域名",
  "server_port": 8443,
  "up_mbps": 50,
  "down_mbps": 200,
  "password": "服务端 auth.password 里的同一个密码",
  "tls": {
    "enabled": true,
    "server_name": "www.cloudflarestatus.com",
    "insecure": true
  }
}

如果服务端开启了 Salamander 混淆，sing-box 客户端要加入同样的 obfs：

{
  "type": "hysteria2",
  "tag": "hy2-freebsd-user",
  "server": "你的服务器IP或域名",
  "server_port": 8443,
  "up_mbps": 50,
  "down_mbps": 200,
  "password": "服务端 auth.password 里的同一个密码",
  "obfs": {
    "type": "salamander",
    "password": "服务端 obfs.salamander.password 里的同一个密码"
  },
  "tls": {
    "enabled": true,
    "server_name": "www.cloudflarestatus.com",
    "insecure": true
  }
}

如果服务端没有开启 obfs，客户端也不要写 obfs。两边必须一致。

up_mbps 和 down_mbps 是客户端告诉 Hysteria2 的可用带宽。按自己的实际线路填写，不要照抄 1000。

12. DNS 解析可选配置

默认情况下可以先不写 resolver。如果 Hysteria2 出站访问时经常出现解析异常或 i/o timeout，再考虑改成 DoH。

DoH resolver 示例：

resolver:
  type: https
  strategy: prefer_ipv4
  https:
    addr: 1.1.1.1:443
    sni: cloudflare-dns.com
    timeout: 5s

保存后重启：

~/bin/hctl restart
~/bin/hctl log

13. 系统参数优化

普通用户版也可以受益于 FreeBSD 的 UDP/QUIC 系统参数优化，但普通用户通常没有权限修改 /etc/sysctl.conf 和 /boot/loader.conf。

这一节推荐但非必需。如果只是先跑通服务，可以先跳过；如果是长期运行、追求速度，或者服务器带宽比较高，建议让 root 或管理员按 root 版文章里的系统参数优化执行。

它的作用不是凭空提速，而是减少高吞吐 UDP/QUIC 场景下因为系统缓冲区偏小造成的丢包、抖动和不稳定。Hysteria2 应用层的 QUIC 窗口先保持默认即可，不建议一上来就乱调。

14. 升级 Hysteria2

后续想升级时，先打开 Hysteria2 官方 Releases 页面，确认最新 tag。例如当前版本是 app/v2.9.2，如果以后看到 app/v2.9.3 或更高版本，就把下面的 HY2_VERSION 改掉。

先备份当前二进制和配置：

cp ~/.usr/local/bin/hysteria ~/.usr/local/bin/hysteria.bak.$(date +%Y%m%d%H%M%S)
cp ~/.usr/local/etc/hysteria/config.yaml ~/.usr/local/etc/hysteria/config.yaml.bak.$(date +%Y%m%d%H%M%S)

停止服务：

~/bin/hctl stop

下载并覆盖新版：

HY2_VERSION='app/v2.9.2'
fetch -o /tmp/hysteria "https://github.com/apernet/hysteria/releases/download/${HY2_VERSION}/hysteria-freebsd-amd64"
install -m 755 /tmp/hysteria ~/.usr/local/bin/hysteria
rm -f /tmp/hysteria

检查版本和配置：

~/.usr/local/bin/hysteria version
~/.usr/local/bin/hysteria server -c ~/.usr/local/etc/hysteria/config.yaml

如果前台运行没有报错，按 Ctrl+C 退出，再启动服务：

~/bin/hctl start
~/bin/hctl status
~/bin/hctl log

如果新版启动失败，可以临时回滚旧二进制：

~/bin/hctl stop
cp ~/.usr/local/bin/hysteria.bak.时间戳 ~/.usr/local/bin/hysteria
~/bin/hctl start

15. 清理环境并重启

如果之前配置错过 DNS 或证书路径，可以先清理旧进程和旧日志：

~/bin/hctl stop
pkill -u "$USER" hysteria
> ~/.usr/local/etc/hysteria/hysteria.log
~/bin/hctl start
~/bin/hctl log

如果你明确知道用户名，也可以写成：

pkill -u felix hysteria

完成

到这里，普通用户权限下的 Hysteria2 服务端就可以通过 tmux 后台运行，并通过 crontab 在开机后自动拉起。

日常维护主要看四件事：

  • ~/.usr/local/etc/hysteria/config.yaml
  • ~/.usr/local/etc/hysteria/hysteria.log
  • ~/bin/hctl status
  • 云服务器和 FreeBSD 防火墙是否放行 UDP 8443
