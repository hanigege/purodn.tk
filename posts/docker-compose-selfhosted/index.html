Docker Compose 部署自托管服务的基本规范

目录、数据卷、环境变量和反向代理先规范，后期维护会轻松很多。

用 Docker Compose 部署服务很快，但如果目录和配置随手放，几个月后维护会非常痛苦。

一个服务一个目录

建议每个服务独立目录，例如 /opt/stacks/immich、/opt/stacks/navidrome。目录里保留 compose.yml、.env 和必要说明。

数据卷要明确

重要数据尽量绑定到宿主机明确路径，不要只依赖匿名卷。备份时你应该能一眼看出哪些目录必须保留。

环境变量单独管理

密码、域名、端口等变量放进 .env。公开分享配置时，不要把真实密钥和 Token 一起发出去。

反向代理统一入口

多个 Web 服务建议通过 Caddy、Nginx Proxy Manager 或 Traefik 统一管理 HTTPS 和域名，避免每个容器各自暴露公网端口。
