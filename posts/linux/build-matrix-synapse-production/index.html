从零搭建 Matrix Synapse 生产环境 —— 小白手搓教程（内网非标端口）

一台服务器、一个域名、跟着本文一步步配出完整的 Matrix 聊天服务（含语音视频通话）

前言

为什么你需要这个教程？

你在网上看到的 Matrix 教程大多只教你跑起一个 Synapse 容器就完了。但现实中你要面对：反向代理、SSL 证书、数据库、TURN 服务器、语音视频通话（LiveKit）、域名发现（.well-known）、端口被封的各种骚操作。本文就是照着我们的生产环境一步步整理出来的，你照做就能得到一套和我们现在用的一模一样的完整部署。

本教程特色：

  • ✅ 全实战 —— 所有配置直接从生产服务器原文复制，没有魔改
  • ✅ 小白友好 —— 从「去名字商买域名」开始说，每个命令都解释干什么用
  • ✅ 域名可替换 —— 每处用到域名的位置都会有 💡 提示，你用自己域名时知道改哪里
  • ✅ 密码自动生成 —— 所有密钥密码都给出生成命令，不会让你想破头
  • ✅ 完整栈 —— 不只是聊天，语音视频通话、管理后台、通知推送、动态域名全部配齐

你需要准备的东西

硬件

项目 最低要求 建议
CPU 2 核 4 核

系统 Debian 12 / Ubuntu 22.04+ Debian 12

这是给 5~10 人小家庭/团队用的。人多了再往上加配置。

A element 同上

💡 换域名提示：全文凡是出现 116160.xyz 的地方，替换成 你的域名 即可。我会在每个出现的地方标注 💡。

端口

如果你的服务器 80/443 端口没有被封，那最省事。但很多家庭宽带会封 443。我们的生产机就遇到这个问题，解决方案如下：

  • 被封：用 Cloudflare Workers 做 .well-known 发现，实际流量走非标端口（如 12335、12336）
  • 没被封：跳过 CF Workers 部分，直接用 NPM（Nginx Proxy Manager）处理 80/443

💡 本文按「443 被封，用 CF Workers + 高端口」的方案写。你的 443 没被封的话，后续端口映射直接映射到 80/443 即可。

第一章：服务器基础环境

我们选择 Docker Compose 来管理所有服务。每个服务一个独立容器，互不干扰，更新升级不影响其他服务。

1.1 连接服务器

ssh root@你的服务器IP

首次连接会提示确认指纹，输入 yes 回车。

1.2 安装 Docker

Docker 官方提供一键安装脚本：

curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

装完后验证：

docker --version
# 应该输出 Docker version 2x.xx.x

1.3 安装 Docker Compose

现代 Docker 已经内置了 Compose 插件，验证一下：

docker compose version
# 应该输出 Docker Compose version v2.x.x

如果你的系统比较旧，可能需要安装 docker-compose-plugin 包：apt install docker-compose-plugin

1.4 创建项目文件夹

我们所有服务放在 /root/data/docker_data/ 下，每个服务一个子目录：

mkdir -p /root/data/docker_data/{synapse,postgres16,coturn,livekit-server,matrix-livekit-jwt-service,element-call,npm,synapse-admin,ntfy,ddns-go,matrix-admin-tools,matrix-musicbot,web-call}

💡 文件名说明：以后所有配置文件都放在对应的子目录里，一目了然。

1.5 创建 Docker 网络

多个容器之间需要互相通信，我们创建一个共享网络：

docker network create matrix

这个网络会被 Synapse、PostgreSQL、LiveKit、JWT 服务等容器共用，它们之间通过容器名（而不是 IP）互相访问。

第二章：Nginx Proxy Manager（反向代理）

为什么需要 NPM？

  • 统一的 SSL 证书管理（申请、续期全自动）
  • 域名级别的反向代理（matrix.116160.xyz 转发到 Synapse，element.116160.xyz 转发到 Element）
  • 图形化管理界面，不用手写 Nginx 配置

2.1 docker-compose.yml

创建 /root/data/docker_data/npm/docker-compose.yml：

services:
  nginx-proxy-manager:
    image: 'jc21/nginx-proxy-manager:latest'
    container_name: nginx-proxy-manager
    restart: always
    ports:
      - '80:80'     # HTTP
      - '81:81'     # 管理面板
      - '443:443'   # HTTPS
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt
    networks:
      - matrix
      - npm

networks:
  matrix:
    name: matrix
    external: true
  npm:
    name: npm

💡 注意端口：我们生产机因为 443 被封，NPM 的 443 端口实际映射到宿主机的 12335 端口（见后面 RouterOS 端口转发章节）。如果你的 443 没被封，这段不需要改。

2.2 启动 NPM

cd /root/data/docker_data/npm
docker compose up -d

等几秒后访问 http://你的服务器IP:81，默认登录凭据：

Email:    admin@example.com
Password: changeme

第一次登录会提示修改邮箱和密码，按自己喜好设置。

2.3 申请 SSL 证书

在 NPM 管理面板中：

  1. 点击 SSL Certificates → Add SSL Certificate → Let’s Encrypt
  2. 填写你的域名（比如 matrix.116160.xyz）
  3. 勾选 Force SSL、I Agree
  4. 点击 Save

重复此步骤为每个子域名申请证书（matrix、element、call、livekit、matrix-jwt、coturncoturn7829、synapse-admin）。

💡 换域名提示：把上面所有 116160.xyz 替换成你的域名，子域名部分保持不变。

2.4 配置反向代理规则

在 NPM 面板中点击 Proxy Hosts → Add Proxy Host，为每个服务创建规则：

域名 转发目标 说明
matrix.116160.xyz http://matrix-synapse:8008 Matrix 主服务
element.116160.xyz http://element-web:80 Element Web 客户端
call.116160.xyz http://element-call:80 Element Call 通话
livekit.116160.xyz http://livekit-server:7880 LiveKit 控制面
matrix-jwt.116160.xyz http://matrix-livekit-jwt-service:8191 JWT Token 签发
coturncoturn7829.116160.xyz NPM 不处理这个 TURN 直接走端口
synapse-admin.116160.xyz http://synapse-admin:80 管理面板

💡 换域名提示：域名填你的子域名（如 matrix.你的域名），转发目标里的容器名不要改。

每个规则创建时勾选 SSL → Force SSL，选择你之前申请好的证书。

⚠️ 端口映射说明：这些反代规则都走 NPM 的 443 端口。你的生产机上，NPM 的 443 端口会通过不同方式暴露到公网（见后面端口映射章节）。

第三章：PostgreSQL 数据库

Synapse 需要数据库来存储房间、用户、消息等数据。我们单独起一个 PostgreSQL 16 容器。

3.1 docker-compose.yml

创建 /root/data/docker_data/postgres16/docker-compose.yml：

services:
  postgres16:
    image: postgres:16-alpine
    container_name: postgres16
    restart: always
    shm_size: 256mb
    volumes:
      - ./pgdata:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    environment:
      POSTGRES_PASSWORD: "你的数据库密码"
      POSTGRES_DB: synapse
      TZ: Asia/Shanghai
    networks:
      - matrix

networks:
  matrix:
    name: matrix
    external: true

3.2 创建初始化脚本

创建 /root/data/docker_data/postgres16/init.sql：

-- 创建 Synapse 专用数据库用户
CREATE USER synapse_user WITH PASSWORD '你的Synapse密码';  
-- 创建 Synapse 数据库（所有者是 synapse_user）
CREATE DATABASE synapse OWNER synapse_user;
-- 更改默认搜索引擎（后面会用到）

💡 密码与密钥生成：所有需要密码的地方，统一用这个命令生成随机密码：

openssl rand -base64 32
# 输出类似：x7Kq3mP8nR2tV5wY9bE1hJ4sL6uA0cFd

或者你可以用简单点的：

date +%s | sha256sum | base64 | head -c 32

3.3 初始化 SQL 脚本

创建 /root/data/docker_data/postgres16/init.sql：

CREATE USER synapse_user WITH PASSWORD '这里填你的密码';
CREATE DATABASE synapse OWNER synapse_user;
GRANT ALL PRIVILEGES ON DATABASE synapse TO synapse_user;

用上面 openssl rand -base64 32 生成的密码替换 这里填你的密码。

3.4 启动 PostgreSQL

cd /root/data/docker_data/postgres16
docker compose up -d

验证：

docker logs postgres16 | tail -5
# 应该看到 "database system is ready to accept connections"

第四章：Matrix Synapse 核心服务

4.1 docker-compose.yml

创建 /root/data/docker_data/synapse/docker-compose.yml：

    image: matrixdotorg/synapse:latest
    container_name: matrix-synapse

端口说明：8008 是 Synapse 的内部 HTTP 端口。它不直接暴露到公网，而是通过 NPM 反向代理出去。

4.2 生成配置文件

先创建数据目录并生成初始配置：

mkdir -p /root/data/docker_data/synapse/data

  -v /root/data/docker_data/synapse/data:/data \
  -e SYNAPSE_SERVER_NAME=你的域名 \
  -e SYNAPSE_REPORT_STATS=yes \
  matrixdotorg/synapse:latest generate

💡 换域名提示：把 你的域名 替换成你的主域名（比如 116160.xyz），这是你的 Matrix 服务标识（server_name）。

生成后你会看到 data/ 目录下多了 homeserver.yaml、你的域名.signing.key、你的域名.log.config 等文件。

4.3 编辑 homeserver.yaml

现在我们来修改 homeserver.yaml，这是 Synapse 最核心的配置文件。用你喜欢的编辑器打开：

nano /root/data/docker_data/synapse/data/homeserver.yaml

下面是完整的、生产可用的配置（逐段解释）：

核心标识

server_name: "你的域名"
pid_file: /data/homeserver.pid
public_baseurl: "https://matrix.你的域名:12335"

💡 换域名提示：server_name 填你的主域名，public_baseurl 填 https://matrix.你的域名:12335（端口 12335 是因为我们 443 被封，你的 443 没被封的话改成 https://matrix.你的域名）

为什么 public_baseurl 要带端口？ 因为我们的 ISP 封了 443，所以实际入口是 12335。Clients 连接 Matrix 时会用这个 URL。如果你的 443 没封，就写成 https://matrix.你的域名（不加端口）。

联邦（Federation）

federation_domain_whitelist:
  - "你的域名"
  # 允许从这些域名拉取贴纸包媒体
  - "t2bot.io"
  - "matrix.org"

联邦白名单只允许和你的域名通信。t2bot.io 和 matrix.org 是为了能显示从这些服务器发出的贴纸图片。

监听配置

listeners:
  - port: 8008
    tls: false
    type: http
    x_forwarded: true
    resources:
      - names: [client, federation]
        compress: false
  - port: 8448
    tls: false
    type: http
    x_forwarded: true
    resources:
      - names: [client, federation]
        compress: false

8008 是主端口，8448 是联邦备用端口。x_forwarded: true 告诉 Synapse 信任 NPM 传递的真实客户端 IP。

数据库连接

database:
  name: psycopg2
  args:
    user: synapse_user
    password: 你的Synapse数据库密码
    database: synapse
    host: postgres16

    cp_min: 5

4.6 配置客户端发现

为了让 Matrix 客户端（Element）知道你 Synapse 服务器的地址，需要配置 /.well-known/matrix/client。

方式 A：443 没被封 —— 直接用 Web 服务器

在 NPM 或你的 Web 服务器上提供静态文件。我们因为 443 被封，用的是方式 B。

方式 B：443 被封 —— Cloudflare Workers（推荐）

创建一个 Cloudflare Worker，路由设置为 你的域名/.well-known/matrix/*，代码如下：

const homeserverUrl = "https://matrix.你的域名:12335";
const jwtUrl = "https://matrix-jwt.你的域名:12336";
const federationServer = "matrix.你的域名:12335";

function json(data) {
  return new Response(JSON.stringify(data), {
    headers: {
      "content-type": "application/json; charset=utf-8",
      "access-control-allow-origin": "*",
      "access-control-allow-methods": "GET, OPTIONS",
      "cache-control": "no-store, no-cache, must-revalidate, proxy-revalidate, max-age=0",
      "pragma": "no-cache",
      "expires": "0"
    }
  });
}

export default {
  async fetch(request) {
    const url = new URL(request.url);

    if (request.method === "OPTIONS") {
      return new Response(null, {
        status: 204,
        headers: {
          "access-control-allow-origin": "*",
          "access-control-allow-methods": "GET, OPTIONS"
        }
      });
    }

    if (url.pathname === "/.well-known/matrix/client") {
      return json({
        "m.homeserver": {
          "base_url": homeserverUrl
        },
        "org.matrix.msc4143.rtc_foci": [
          {
            "type": "livekit",
            "livekit_service_url": jwtUrl
          }
        ],
        "io.element.rtc": {
          "livekit": {
            "service_url": jwtUrl
          }
        }
      });
    }

    if (url.pathname === "/.well-known/matrix/server") {
      return json({
        "m.server": federationServer
      });
    }

    return new Response("Not Found", { status: 404 });
  }
};

💡 换域名提示：把上面代码中所有 你的域名 换成你自己的域名。

Worker 路由设置：你的域名/.well-known/matrix/* → 选择这个 Worker。

这个 Worker 做了什么？

  • 客户端访问 https://116160.xyz/.well-known/matrix/client 时，返回 JSON 告诉客户端 Matrix 服务器在 matrix.116160.xyz:12335，LiveKit 服务在 matrix-jwt.116160.xyz:12336
  • 其他 Federation 服务器访问 .well-known/matrix/server 时，告知联邦入口
  • 因为 Worker 跑在 Cloudflare 的 443 端口上（CF 自己就有 443），所以即使你的服务器 443 被封，这个发现文件也能正常访问

第六章：TURN/STUN 服务器（coturn）

TURN/STUN 服务器用于语音视频通话时的网络穿透。当用户之间无法直接建立 P2P 连接时（比如 NAT 网络环境），通过 TURN 服务器中继数据。如果只能文字聊天，不需要语音视频通话，可以跳过这一章。

6.1 docker-compose.yml

创建 /root/data/docker_data/coturn/docker-compose.yml：

services:
  coturn:
    image: instrumental/coturn:latest
    container_name: coturn
    restart: always
    network_mode: "host"
    volumes:
      - ./turnserver.conf:/etc/turnserver.conf
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

networks: {}

为什么用 network_mode: "host"？ TURN 需要操作大量 UDP 端口（49152-49200），用 host 网络模式可以避免 Docker 的端口映射瓶颈。

6.2 配置文件

创建 /root/data/docker_data/coturn/turnserver.conf：

# TURN/STUN 服务器配置
listening-device=eth0
listening-port=3478
tls-listening-port=5349
#中继端口范围
min-port=49152
max-port=49200
# 域名
realm=coturncoturn7829.你的域名
# 认证方式
use-auth-secret
static-auth-secret=你的TURN共享密钥
# 允许从任意 IP 连接
no-auth

# 安全配置
fingerprint
lt-cred-mech
# 日志
Verbose

# 支持证书 (如果配置了 Let's Encrypt)
cert=/etc/letsencrypt/live/coturncoturn7829.你的域名/fullchain.pem
pkey=/etc/letsencrypt/live/coturncoturn7829.你的域名/privkey.pem

💡 换域名提示：realm 和证书路径中的 你的域名 替换成你的域名。

static-auth-secret 用 openssl rand -base64 32 生成，这个值必须和 Synapse 的 homeserver.yaml 中的 turn_shared_secret 保持一致！

6.3 SSL 证书

因为 coturn 用了 host 网络模式，不能直接用 Docker 网络里的 NPM 管理证书。你需要手动把 NPM 申请的证书复制过来：

# 先从 NPM 的数据目录找到证书
ls /root/data/docker_data/npm/letsencrypt/live/
# 找到 coturncoturn7829.你的域名 目录，复制到 coturn 目录下
cp -r /root/data/docker_data/npm/letsencrypt/ /root/data/docker_data/coturn/letsencrypt

6.4 启动

cd /root/data/docker_data/coturn
docker compose up -d

验证：

docker logs coturn | tail -5
# 应该看到 "TURN/STUN server started"

Dockerfile（放在同目录下）：

WORKDIR /app

    flask \
    livekit-api \
    requests

COPY app.py .

EXPOSE 8191

CMD ["python3", "app.py"]

核心逻辑 app.py（在生产机上运行的文件）：

import os
import json
import requests
from flask import Flask, request, jsonify
from livekit.api import AccessToken, VideoGrant

app = Flask(__name__)

LIVEKIT_API_KEY = os.environ.get("LIVEKIT_API_KEY")
LIVEKIT_API_SECRET = os.environ.get("LIVEKIT_API_SECRET")
LIVEKIT_HOST = os.environ.get("LIVEKIT_HOST", "livekit-server")
LIVEKIT_PORT = os.environ.get("LIVEKIT_PORT", "7880")
SYNAPSE_SERVER = os.environ.get("SYNAPSE_SERVER", "matrix-synapse")
SYNAPSE_PORT = os.environ.get("SYNAPSE_PORT", "8008")
JWT_SECRET = os.environ.get("JWT_SECRET")

@app.route("/")
def index():
    return jsonify({"status": "ok", "service": "matrix-livekit-jwt"})

@app.route("/livekit/join", methods=["POST"])
def join_call():
    data = request.get_json()
    if not data or "user_id" not in data:
        return jsonify({"error": "missing user_id"}), 400

    user_id = data["user_id"]
    room_name = data.get("room_name", "default")

    # 验证用户身份（向 Synapse 确认用户是否存在）
    try:
        resp = requests.get(
            f"http://{SYNAPSE_SERVER}:{SYNAPSE_PORT}/_matrix/client/v3/profile/{user_id}",
            timeout=5
        )
        if resp.status_code != 200:
            return jsonify({"error": "user not found"}), 401
    except Exception as e:
        return jsonify({"error": f"synapse error: {str(e)}"}), 502

    # 生成 LiveKit 访问令牌
    token = AccessToken(LIVEKIT_API_KEY, LIVEKIT_API_SECRET)
    token.identity = user_id
    token.add_grant(VideoGrant(room_join=True, room=room_name))
    jwt = token.to_jwt()

    return jsonify({
        "token": jwt,
        "livekit_host": LIVEKIT_HOST,
        "livekit_port": LIVEKIT_PORT,
        "room": room_name
    })

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8191, debug=False)
