从零搭建 Matrix Synapse 私有聊天服务器（443标准版）

标准网络版 — 基于 Docker 部署完整 Matrix 栈（Synapse + Caddy + PostgreSQL + LiveKit 通话 + Element），全程可操作可复制，小白也能一步步搭起来

这是一份面向零基础小白的教程，手把手教你从建文件夹开始，把一套完整的 Matrix 聊天服务器跑起来。你不需要懂 Linux 内核、不需要懂网络协议、不需要背命令——跟着复制粘贴就行。

本教程基于生产环境的真实配置整理，所有服务都用 Docker 部署，方便维护和迁移。

一、准备工作

1.1 服务器要求

  • CPU：1 核起步，2 核流畅
  • 内存：2GB（4GB 更佳，群聊多就加）
  • 硬盘：20GB（图片文件会随使用增长）
  • 系统：Ubuntu 22.04 / Debian 12（其他 Linux 也行，命令微调）
  • 网络：公网 IP，防火墙放开以下端口
端口 协议 用途
443 TCP HTTPS 网页访问
8448 TCP Matrix 联邦通信
3478 UDP TURN 语音视频
5349 TCP/TLS TURN 加密连接
7880 TCP LiveKit API
17882 TCP LiveKit TCP 通道
50201-50400 UDP LiveKit 媒体流

不搞联邦可以只开 443。本教程按完整配置来，不用的端口不开也不影响。

1.2 域名规划

你需要一个根域名 + 7 个子域名（可选服务可以跳过），全部解析到服务器 IP。

域名 用途 必选
example.com 根域 — well-known 配置 + 自动跳转 ✅
matrix.example.com Matrix 客户端 API + 联邦入口 ✅
element.example.com Element Web 聊天界面 ✅
call.example.com Element Call 语音通话 ✅ 要通话就必选
livekit.example.com LiveKit 通话服务器 ✅ 同上
matrix-jwt.example.com 通话令牌服务 ✅ 同上
synapse-admin.example.com 管理后台 可选
ntfy.example.com 推送通知 可选

全部添加 A 记录，指向你的服务器公网 IP。担心不封端口就别开 8448 的联邦端口。

1.3 要准备的东西

  • ✅ 一台服务器（本文以 Debian 12 为例）
  • ✅ 一个域名（本文以 example.com 为例，请全部替换为你的真实域名）
  • ✅ SSH 客户端（Windows 用 Terminal/PuTTY，Mac/Linux 直接终端）

1.4 前置知识

只需要两个概念：

  • SSH：远程登录服务器的方式，像远程桌面但只有黑框
  • Docker：一键打包运行软件的容器技术，类似于给每个软件一个独立的小盒子

记住了，下面全是复制粘贴。

二、服务器初始化

2.1 SSH 登录服务器

在自己的电脑上打开终端（Windows 用 PowerShell 或 Terminal），输入：

ssh root@你的服务器IP

初次登录会提示确认指纹，输入 yes 回车，然后输入密码。

2.2 安装 Docker

逐条执行，一条一行：

curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

等跑完，验证安装：

docker --version
docker compose version

两条都有版本号输出就对了。

2.3 创建 Docker 网络

所有服务都跑在同一个 Docker 内部网络里，方便它们互相通信：

docker network create matrix

这个网络后面所有 docker-compose.yml 里都会用到，名字固定为 matrix。

2.4 建文件夹

mkdir -p /root/data/docker_data
cd /root/data/docker_data

一个文件夹放一个服务，清清楚楚。后面所有的 docker-compose.yml 都在 /root/data/docker_data/ 下的子目录里。

三、PostgreSQL 数据库

Synapse 支持两种数据库：SQLite（默认）和 PostgreSQL。SQLite 适合测试，PostgreSQL 性能好很多、支持并发访问，正式用强烈推荐。

3.1 创建数据库目录和配置文件

mkdir -p /root/data/docker_data/synapse/postgres
cd /root/data/docker_data/synapse

创建 docker-compose.yml：

services:
  postgres16:
    image: postgres:16-alpine
    container_name: postgres16
    restart: unless-stopped
    ports:
      - '127.0.0.1:5432:5432'
    volumes:
      - ./postgres:/var/lib/postgresql/data
    networks:
      - matrix
    environment:
      POSTGRES_PASSWORD: change_me_plz
      POSTGRES_USER: synapse
      POSTGRES_DB: synapse
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U synapse"]
      interval: 5s
      timeout: 5s
      retries: 5

networks:
  matrix:
    name: matrix
    external: true

127.0.0.1:5432:5432 表示 PostgreSQL 只监听本机（不对外暴露），通过 Docker 内部网络 matrix 供 Synapse 连接。change_me_plz 请替换成随机密码。

3.2 启动数据库

docker compose up -d

等几秒检查状态：

docker compose ps

看到 postgres16 的 STATUS 是 healthy 就说明启动好了。

四、Synapse 核心服务器

4.1 生成 Synapse 配置

docker run -it --rm \
  -v /root/data/docker_data/synapse/data:/data \
  -e SYNAPSE_SERVER_NAME=example.com \
  -e SYNAPSE_REPORT_STATS=no \
  matrixdotorg/synapse:latest generate

把 example.com 替换成你的根域名。这条命令会生成 homeserver.yaml 和几个密钥文件到 ./data 文件夹。

等命令跑完，检查文件：

ls -la /root/data/docker_data/synapse/data/

应该能看到 homeserver.yaml、example.com.signing.key、example.com.log.config 等文件。

4.2 修改 homeserver.yaml

nano /root/data/docker_data/synapse/data/homeserver.yaml

找到对应的行，改成下面的内容（没有的行就手动添加）：

# 服务器名称（你的根域名）
server_name: "example.com"

# 公开访问地址
public_baseurl: https://matrix.example.com

# 监听配置 — 两个分开的监听器
listeners:
  # 8008：客户端 API（网页、App 访问）
  - port: 8008
    tls: false
    type: http
    x_forwarded: true
    bind_addresses: ['0.0.0.0']
    resources:
      - names: [client]
        compress: true

  # 8448：联邦通信（和其他 Matrix 服务器通信）
  - port: 8448
    tls: false
    type: http
    x_forwarded: true
    bind_addresses: ['0.0.0.0']
    resources:
      - names: [federation]

# 数据库 — PostgreSQL
database:
  name: psycopg2
  args:
    user: synapse
    password: change_me_plz
    database: synapse
    host: postgres16
    port: 5432
    cp_min: 5
    cp_max: 10

# 文件存储位置
media_store_path: "/data/media_store"
uploads_path: "/data/uploads"

# 最大上传文件 50MB
max_upload_size: 50M

# 注册配置（关闭开放注册，用命令创建用户）
enable_registration: false
registration_shared_secret: "一个随机字符串，跟你密码一样重要"

# 联邦白名单（只和本域通信，防滥用）
federation_domain_whitelist:
  - example.com

# TURN 服务器配置（用于语音视频通话）
turn_uris:
  - "turn:example.com:3478?transport=udp"
  - "turn:example.com:3478?transport=tcp"
turn_shared_secret: "coturn的static-auth-secret，等会配置时用"
turn_allow_guests: true

# 管理 API
admin: true

# 批量注册限制
rc_registration:
  per_second: 0.17
  burst_count: 3

# 实验性功能（通话集成需要）
experimental_features:
  msc4143: true

保存退出（nano 里按 Ctrl+X，按 Y，回车）。

registration_shared_secret 和 turn_shared_secret 需要随机生成，可以用 openssl rand -hex 32 生成。

4.3 创建 Synapse 的 docker-compose.yml

在 /root/data/docker_data/synapse/ 下（注意没有覆盖之前的 postgres16）：

cd /root/data/docker_data/synapse

这个目录现在已经有 docker-compose.yml（含 postgres16），追加 Synapse 服务到同一个文件。用 nano docker-compose.yml 打开，在 services: 下增加：

  synapse:
    image: matrixdotorg/synapse:latest
    container_name: matrix-synapse
    restart: unless-stopped
    ports:
      - '127.0.0.1:8008:8008'
      - '127.0.0.1:8448:8448'
    volumes:
      - ./data:/data
    networks:
      - matrix
    depends_on:
      postgres16:
        condition: service_healthy
    environment:
      SYNAPSE_CONFIG_PATH: /data/homeserver.yaml
      SYNAPSE_SERVER_NAME: example.com
      SYNAPSE_REPORT_STATS: "no"

注意缩进对齐，services: 下面的 postgres16: 和 synapse: 要在同一列。

完整的文件应该像这样：

services:
  synapse:
    image: matrixdotorg/synapse:latest
    container_name: matrix-synapse
    restart: unless-stopped
    ports:
      - '127.0.0.1:8008:8008'
      - '127.0.0.1:8448:8448'
    volumes:
      - ./data:/data
    networks:
      - matrix
    depends_on:
      postgres16:
        condition: service_healthy
    environment:
      SYNAPSE_CONFIG_PATH: /data/homeserver.yaml
      SYNAPSE_SERVER_NAME: example.com
      SYNAPSE_REPORT_STATS: "no"

  postgres16:
    image: postgres:16-alpine
    container_name: postgres16
    restart: unless-stopped
    ports:
      - '127.0.0.1:5432:5432'
    volumes:
      - ./postgres:/var/lib/postgresql/data
    networks:
      - matrix
    environment:
      POSTGRES_PASSWORD: change_me_plz
      POSTGRES_USER: synapse
      POSTGRES_DB: synapse
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U synapse"]
      interval: 5s
      timeout: 5s
      retries: 5

networks:
  matrix:
    name: matrix
    external: true

4.4 启动 Synapse

docker compose up -d

查看日志确认启动正常：

docker compose logs synapse --tail 30

看到 Synapse now listening on port 8008 和 8448 就对了。

五、Caddy 反向代理

Caddy 是目前最友好的反向代理，自动申请和续签 SSL 证书，配置简单，小白福音。

5.1 创建 Caddy 目录和配置

mkdir -p /root/data/docker_data/caddy
cd /root/data/docker_data/caddy

创建 docker-compose.yml：

services:
  caddy:
    image: caddy:latest
    container_name: caddy
    restart: unless-stopped
    network_mode: host
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
      - caddy_config:/config

volumes:
  caddy_data:
  caddy_config:

network_mode: host 让 Caddy 直接监听宿主机端口（443/80），不走 Docker 网络转发，性能最好。Caddy 通过 matrix 网络的 DNS 名称（如 synapse:8008）访问其他容器。

5.2 编写 Caddyfile

创建 Caddyfile（没有后缀名）：

{
  email admin@example.com
}

# 根域 — well-known + 跳转到 Element
example.com {
  # well-known 由 Caddy 直接返回静态 JSON
  @wk_server path /.well-known/matrix/server
  @wk_client path /.well-known/matrix/client
  handle @wk_server {
    header Content-Type application/json
    respond `{"m.server": "matrix.example.com:443"}`
  }
  handle @wk_client {
    header Content-Type application/json
    respond `{"m.homeserver": {"base_url": "https://matrix.example.com"}}`
  }

  # 其余请求跳转到 Element Web
  redir https://element.example.com{uri} permanent
}

# Element Web 前端
element.example.com {
  reverse_proxy element-web:80
}

# Element Call 语音通话
call.example.com {
  reverse_proxy element-call:80
}

# Matrix 客户端 API（端口 443）
matrix.example.com {
  header {
    Access-Control-Allow-Origin *
    Access-Control-Allow-Methods "GET POST PUT DELETE OPTIONS"
    Access-Control-Allow-Headers "X-Requested-With, Content-Type, Authorization, Date"
  }

  # MSC4143 — 通话能力发现
  handle /_matrix/client/v1/rtc/transports {
    header Content-Type application/json
    header Access-Control-Allow-Origin *
    respond `[{"url": "https://matrix-jwt.example.com", "type": "livekit"}]`
  }

  # CORS 预检请求
  @options method OPTIONS
  handle @options {
    header Access-Control-Allow-Origin *
    header Access-Control-Allow-Methods "GET POST PUT DELETE OPTIONS"
    header Access-Control-Allow-Headers "X-Requested-With, Content-Type, Authorization, Date"
    header Access-Control-Max-Age 86400
    respond 204
  }

  reverse_proxy synapse:8008
}

# Matrix 联邦入口（端口 8448）
matrix.example.com:8448 {
  reverse_proxy synapse:8448
}

# LiveKit 服务
livekit.example.com {
  header Access-Control-Allow-Origin *
  reverse_proxy 127.0.0.1:7880
}

# LiveKit JWT 令牌服务
matrix-jwt.example.com {
  reverse_proxy 127.0.0.1:8170
}

# Synapse Admin 管理面板
synapse-admin.example.com {
  reverse_proxy synapse-admin:80
}

# ntfy 推送通知
ntfy.example.com {
  header Access-Control-Allow-Origin *
  reverse_proxy ntfy:80
}

全部 example.com 替换成你的真实域名。admin@example.com 改成你的邮箱，用于 Let’s Encrypt 证书通知。

5.3 启动 Caddy

docker compose up -d

查看日志：

docker compose logs --tail 20

看到 successfully obtained certificate 说明 SSL 证书自动申请成功了。等 1-2 分钟，DNS 生效后 Caddy 就能自动搞定证书。

验证 well-known 是否生效：

curl -s https://example.com/.well-known/matrix/client

应该返回：

{"m.homeserver": {"base_url": "https://matrix.example.com"}}

六、创建管理员用户

Synapse 默认关闭开放注册，需要用命令行创建用户。

进入 Synapse 容器：

docker exec -it matrix-synapse register_new_matrix_user \
  -u admin \
  -p 你的密码 \
  -c /data/homeserver.yaml \
  http://localhost:8008
  • admin：用户名（可自定义）
  • 你的密码：请改成高强度密码
  • 系统会问 Make admin? [no]，输入 yes 回车

如果提示 M_UNKNOWN 错误，先确认 Synapse 是否运行正常：docker ps | grep matrix-synapse。

七、Element Web 前端

7.1 创建 Element Web 目录

mkdir -p /root/data/docker_data/element-web
cd /root/data/docker_data/element-web

7.2 docker-compose.yml

services:
  element-web:
    image: ghcr.io/element-hq/element-web:latest
    container_name: element-web
    restart: unless-stopped
    volumes:
      - ./config.json:/app/config.json

    name: matrix

不需要暴露端口，Element Web 通过 Caddy 反代访问，Caddy 通过 Docker 内部网络直接连接 element-web:80。

7.3 config.json

{
  "default_server_config": {
    "m.homeserver": {
      "base_url": "https://matrix.example.com"
    },
    "m.identity_server": {
      "base_url": "https://matrix.example.com"
    }

  "disable_guests": true,
  "features": {
    "element_call_video_rooms": true
  },
  "element_call": {
    "url": "https://call.example.com",
    "use_exclusively": true
  }
}

base_url 和 url 中的 example.com 都要替换成你的真实域名。

7.4 启动

docker compose up -d

等几秒，访问 https://element.example.com 应该能看到登录页面。

八、TURN/STUN 服务器（Coturn）

语音视频通话需要 TURN 服务器做中继。当两个人不能直连时（NAT 限制），TURN 负责转发音视频流。

8.1 创建目录和配置

mkdir -p /root/data/docker_data/coturn/etc/coturn
cd /root/data/docker_data/coturn

创建 etc/coturn/turnserver.conf：

listening-port=3478
tls-listening-port=5349
listening-ip=0.0.0.0
relay-ip=0.0.0.0
external-ip=你的服务器公网IP

fingerprint
lt-cred-mech
use-auth-secret
static-auth-secret=用 openssl rand -hex 32 生成

realm=example.com
total-quota=100
bps-capacity=0
stale-nonce=600

no-tlsv1
no-tlsv1_1
no-cli
no-stdout-log

static-auth-secret 的值必须和 homeserver.yaml 里的 turn_shared_secret 一致。用命令生成：openssl rand -hex 32。

8.2 docker-compose.yml

services:
  coturn:
    image: coturn/coturn:latest
    container_name: coturn
    restart: unless-stopped
    network_mode: host
    command:
      - "-c"
      - "/etc/coturn/turnserver.conf"
      - "--log-file=stdout"
    volumes:
      - ./etc/coturn/turnserver.conf:/etc/coturn/turnserver.conf

8.3 启动

docker compose up -d

检查日志：

docker compose logs --tail 10

看到 listening on ... :3478 就对了。

九、LiveKit 通话栈

Element Call 使用 LiveKit 作为通话引擎。整个通话栈包含三个服务：

  1. LiveKit 服务器 — 核心通话引擎
  2. LiveKit JWT 服务 — 生成通话令牌
  3. Element Call — 通话前端界面

9.1 LiveKit 服务器

创建目录

mkdir -p /root/data/docker_data/livekit-server/etc
cd /root/data/docker_data/livekit-server

livekit.yaml

port: 7880
rtc:
  tcp_port: 17882
  port_range_start: 50201
  port_range_end: 50400
  use_external_ip: true
  stun_servers:
    - stun:example.com:3478
keys:
  apikey: "用 openssl rand -hex 32 生成"
turn:
  enabled: true
  domain: example.com
  cert_file: /etc/livekit/coturn.pem
  key_file: /etc/livekit/coturn-key.pem

keys.apikey 的值需要记住，后面 JWT 服务和 Caddy 配置里都要用到。stun_servers 指向你的 Coturn 地址。

由于 Coturn 使用 use-auth-secret 模式，LiveKit 需要 Coturn 的证书来做 TURN 集成。这里简单起见可以先不配 TURN 集成（对通话影响不大），把 turn 部分注释掉：

port: 7880
rtc:
  tcp_port: 17882
  port_range_start: 50201
  port_range_end: 50400
  use_external_ip: true
  stun_servers:
    - stun:example.com:3478
keys:
  apikey: "用 openssl rand -hex 32 生成"
# turn:
#   enabled: true
#   domain: example.com

docker-compose.yaml

services:
  livekit:
    image: livekit/livekit-server:latest
    container_name: livekit-server
    restart: unless-stopped
    network_mode: host
    command:
      - --config
      - /etc/livekit.yaml
    volumes:
      - ./etc/livekit.yaml:/etc/livekit.yaml

network_mode: host 让 LiveKit 直接使用宿主机网络，处理音视频流时性能更好。

9.2 LiveKit JWT 服务

Element Call 需要一个令牌服务来获取 LiveKit 的访问令牌。这里使用 Element 官方的 lk-jwt-service。

创建目录

mkdir -p /root/data/docker_data/matrix-livekit-jwt-service
cd /root/data/docker_data/matrix-livekit-jwt-service

docker-compose.yaml

services:
  lk-jwt-service:
    image: ghcr.io/element-hq/lk-jwt-service:latest
    container_name: matrix-livekit-jwt-service
    restart: unless-stopped
    ports:
      - '127.0.0.1:8170:80'
    networks:
      - matrix
    environment:
      LIVEKIT_URL: https://livekit.example.com
      LIVEKIT_API_KEY: apikey
      LIVEKIT_API_SECRET: "上面 livekit.yaml 里生成的密钥"

LIVEKIT_API_KEY 和 LIVEKIT_API_SECRET 要和 livekit.yaml 里的 keys 一致。LIVEKIT_URL 替换你的真实域名。

9.4 启动通话栈

按顺序启动：

cd /root/data/docker_data/livekit-server && docker compose up -d
cd /root/data/docker_data/matrix-livekit-jwt-service && docker compose up -d
cd /root/data/docker_data/element-call && docker compose up -d

检查各容器状态：

docker ps --format 'table {{.Names}}\t{{.Image}}\t{{.Status}}'

全部 Up 就对了。

十、验证

10.1 验证 well-known

curl -s https://example.com/.well-known/matrix/client
curl -s https://example.com/.well-known/matrix/server

应该分别返回 homeserver 信息和联邦地址。

10.2 验证 MSC4143

curl -s https://matrix.example.com/_matrix/client/v1/rtc/transports

应该返回 LiveKit JWT 服务地址。

10.3 登录 Element Web

访问 https://element.example.com，用之前创建的管理员账号登录。

10.4 测试通话

  1. 在 Element 中创建一个房间
  2. 点击右上角电话图标 → 开始通话
  3. 浏览器会请求麦克风/摄像头权限，允许
  4. 如果能听到自己的回音就说明通话功能正常

首次通话可能提示"正在连接"几秒钟，这是 LiveKit 在和 Coturn 协商，正常现象。

11.1 Synapse Admin 管理面板

mkdir -p /root/data/docker_data/synapse-admin

services:

11.2 ntfy 推送通知

让手机 App 能在后台收到消息推送。

mkdir -p /root/data/docker_data/ntfy
cd /root/data/docker_data/ntfy

docker-compose.yml：

services:
  ntfy:
    image: binwiederhier/ntfy
    container_name: ntfy
    restart: unless-stopped
    ports:
      - '127.0.0.1:2586:80'
    volumes:
      - ./data:/var/lib/ntfy

      - serve

push:
  enable_unified_push: true

docker restart matrix-synapse

11.3 矩阵管理工具（matrix-admin-tools）

一站式 Matrix 运维面板，提供媒体清理策略管理、Server Notice 下发、自托管贴纸包管理，并通过 Basic Auth 保护。

前置条件

  • 已完成第 8-9 步的 Synapse + Caddy 部署
  • matrix.jgaga.tk 和 stickers.jgaga.tk 的 DNS A 记录已指向本机 IP

获取源码

cd /root/data/docker_data
git clone https://github.com/hanigege/matrix-manager-ui.git matrix-admin-tools
cd matrix-admin-tools

准备 Secret 文件

mkdir -p /root/data/docker_data/_secrets/synapse-admin
# Synapse Admin API 令牌（从 Synapse 生成）
cat /root/data/docker_data/_secrets/synapse-admin/access_token
# Basic Auth 用户
echo -n "admin" > /root/data/docker_data/_secrets/synapse-admin/admin-user
# Basic Auth 密码（用强随机串）
openssl rand -base64 32 > /root/data/docker_data/_secrets/synapse-admin/admin-password

贴纸导入：如需从 Telegram 导入贴纸包，额外创建：

echo -n "你的TelegramBotToken" > /root/data/docker_data/_secrets/synapse-admin/telegram-bot-token

并在下方 docker-compose.yaml 中取消注释 telegram-bot-token 的 volume 和 TELEGRAM_BOT_TOKEN_FILE 环境变量。

配置 docker-compose.yaml

编辑 docker-compose.yaml，按实际环境修改：

  • SERVER_NAME → 你的主域（如 example.com）
  • STICKER_WIDGET_PUBLIC_URL → https://stickers.你的主域:443/stickerpicker/
  • 如有 LEGACY_STICKER_DB_PATH 指向旧版贴纸数据库可填入，无则留空

参考配置与源码同步存于：GitHub - hanigege/matrix-manager-ui

构建与启动

docker compose build matrix-admin-tools
docker compose up -d

验证运行：

curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8190/
# 应返回 401（无认证时）
curl -s -u admin:$(cat /root/data/docker_data/_secrets/synapse-admin/admin-password) http://127.0.0.1:8190/ | head -3
# 应返回 200 + HTML 页面

Caddy 反向代理

在 Caddyfile 中添加两个站点块：

# 管理面板（需 Basic Auth）
matrix-tools.jgaga.tk {
  encode zstd gzip
  basic_auth {
    admin $2a$...BCRYPT_HASH...
  }
  header Cache-Control "no-store"
  reverse_proxy 127.0.0.1:8190
}

# 贴纸选择器（公开，无需认证）
stickers.jgaga.tk {
  encode zstd gzip
  header Cache-Control "no-store"
  reverse_proxy 127.0.0.1:8190
}

用 caddy hash-password 生成 Bcrypt hash 填入 basic_auth。重载 Caddy：

docker exec caddy caddy reload --config /etc/caddy/Caddyfile

Element 贴纸选择器集成

Element Web 配置中添加 Widget：

字段 值
Widget URL https://stickers.jgaga.tk/stickerpicker/
类型 m.stickerpicker
名称 贴纸

或直接在 config.json 的 widgets 中配置：

"stickerpicker": {
  "type": "m.stickerpicker",
  "url": "https://stickers.jgaga.tk/stickerpicker/",
  "name": "贴纸"
}

rsync -av /root/data/docker_data/matrix-admin-tools/ /root/matrix-manager-ui/ --exclude=data --exclude=logs
git -C /root/matrix-manager-ui add -A && git commit -m "sync: update to production"
