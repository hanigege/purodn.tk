Alpine Linux 将普通用户加入 Docker 组

在 Alpine Linux 上把普通用户加入 docker 组，免 sudo 使用 Docker，并说明 newgrp 不存在时该怎么刷新会话权限。

在 Alpine Linux 上，Docker daemon 通常通过 /var/run/docker.sock 对外提供本机控制入口。这个 socket 一般归属 root:docker，所以普通用户只要加入 docker 组，重新登录后就可以直接执行 docker ps、docker compose up -d 这类命令，不需要每次都加 sudo。

需要注意的是，docker 组等同于本机 root 级别能力：能启动特权容器、挂载宿主机目录、读写系统文件。只应该把可信任的本机用户加入这个组。

结论

推荐命令如下，下面以用户 jay 为例：

rc-update add docker default
rc-service docker start
addgroup jay docker

然后退出当前 SSH 或终端会话，重新登录，再验证：

id
docker ps

如果 id 输出里已经能看到 docker 组，并且 docker ps 不再报权限错误，就说明配置完成。

为什么不推荐写 newgrp docker

很多 Linux 发行版可以用 newgrp docker 在当前 shell 中临时刷新组权限，但 Alpine 最小系统里经常没有这个命令。你看到下面的提示是正常的：

/bin/sh: newgrp: not found

在 Alpine 上最稳的做法是退出当前登录会话，然后重新 SSH 登录。重新登录时，系统会重新加载用户所属组，docker 组权限才会真正进入当前会话。

如果确实想使用 newgrp，通常需要额外安装提供该命令的软件包，例如：

apk add shadow-login
newgrp docker

不同 Alpine 版本里这个命令所属的软件包可能有变化，安装前也可以先搜索：

apk search newgrp

但这不是必须步骤。为了少装额外包，日常服务器上直接重新登录更简单。

完整操作步骤

先确认 Docker 已安装，并且服务存在：

docker --version
rc-service docker status

如果 Docker 服务还没有加入开机启动，可以执行：

rc-update add docker default

启动 Docker 服务：

rc-service docker start

把普通用户加入 docker 组。这里以 jay 为例：

addgroup jay docker

如果当前就是以 jay 登录，也可以写成：

addgroup "$USER" docker

执行完成后，不要只在原来的 shell 里反复测试。请退出当前 SSH：

exit

重新登录后检查当前用户的组：

id

正常情况下，输出里应该能看到类似下面的 docker：

uid=1000(jay) gid=1000(jay) groups=1000(jay),102(docker)

最后验证 Docker 权限：

docker ps

如果能正常显示容器列表，就已经可以免 sudo 使用 Docker。

常见问题

addgroup 命令要不要加 sudo

如果当前已经是 root，不用加 sudo：

addgroup jay docker

如果当前是有 sudo 权限的普通用户，可以加：

sudo addgroup jay docker

Alpine 很多服务器默认直接用 root 管理，未必安装了 sudo。这时照搬 sudo ... 反而会出现 sudo: not found。

docker 组不存在怎么办

正常安装 Docker 包后会创建 docker 组。可以先检查：

getent group docker

如果没有输出，先确认 Docker 是否已安装：

apk add docker
rc-update add docker default
rc-service docker start

然后再执行：

addgroup jay docker

重新登录后还是 permission denied

先看当前会话是否真的拿到了 docker 组：

id

如果没有 docker，说明当前会话还没刷新，或者加入组的用户名不是当前登录用户。确认当前用户名：

whoami

再按真实用户名重新执行：

addgroup "$(whoami)" docker

然后退出并重新登录。

如果 id 里已经有 docker，再检查 Docker socket 权限：

ls -l /var/run/docker.sock

常见正常输出类似：

srw-rw---- 1 root docker 0 Jun  7 21:00 /var/run/docker.sock

如果 Docker 服务没有运行，socket 可能不存在或权限状态不对，重启服务即可：

rc-service docker restart

最小可复制版本

已经确认用户是 jay，并且 Docker 已安装时，只需要这几条：

rc-update add docker default
rc-service docker start
addgroup jay docker
exit

重新登录后：

id
docker ps

不要把 newgrp docker 当成 Alpine 的必需命令。它在很多 Alpine 环境里默认不存在，重新登录才是最通用的刷新方式。
